# 맹세(盟誓) 시스템 구현 지시서

> 이 문서는 **구현 작업자(에이전트 또는 인간)에게 전달되는 지시서**다.
> 시스템 룰과 옵션 수치는 동봉 문서를 참조하고, 본 문서는 **구현 순서·코드 후크 위치·검증 항목**만 다룬다.
>
> 동봉 문서 (먼저 읽을 것):
> - `맹세_시스템_스펙.md` — 시스템 룰·데이터 모델·합산 규칙·캡 함수
> - `맹세_옵션_카탈로그.md` — 16개 옵션 ID/이름/효과/가중치 (확정값)
>
> 작업자는 이 지시서를 읽은 후 **계획부터 세우고 검토 받은 뒤** 코드 변경에 들어간다.

## 0. 절대 원칙

1. **맹세 미사용 시 기존 동작 0% 변화** — 모든 신규 코드는 활성 맹세가 없을 때 단락(short-circuit)되어야 한다. 회귀 테스트 통과 = 본 시스템 구현의 기본 합격선.
2. **율법/철칙(적 측)과 맹세(플레이어 측)는 별도 곱연산** — 두 페널티 레이어를 절대 합치거나 비교하지 말 것. 보상 산정에는 **맹세만** 카운트.
3. **유저 미명시 항목 추가 금지** — 카탈로그에 없는 옵션·효과·수치를 추가하지 말 것. 새 옵션이 필요해 보이면 작업자가 임의 추가 대신 보고할 것.
4. **수치는 카탈로그가 단일 진실 원천** — 코드 상수가 카탈로그와 어긋나면 카탈로그가 맞다.

## 2. 구현 단계 (순서·의존성)

각 단계는 이전 단계 완료를 전제로 한다. 단계마다 PR(또는 커밋) 분리 권장.

### S1. 데이터 모델 (의존성 없음)

**작업**:
1. `data/oaths.ts` 신규 작성

   - `exclusiveGroup` 그룹별 옵션 묶기

**검증**:

- 16개 옵션이 카탈로그와 정확히 일치 (id·name·weight·effect 수치)

### S2. oathSlice — 토글·잠금 액션

**작업**: `store/slices/oathSlice.ts` 신규.

| 액션 | 시그니처 | 동작 |
|------|----------|------|
| `toggleOath` | `(id: string) => void` | activeOathIds에 토글. 같은 `exclusiveGroup`에 다른 옵션이 있으면 자동 해제 (라디오 동작). `lockedAt != null`이면 거부 |
| `lockOathsForField` | `(fieldId: string) => void` | activeOathIds 스냅샷 → `lockedAt = { fieldId, lockedAtTimestamp, snapshotIds }` |
| `unlockOaths` | `() => void` | `lockedAt = null` |
| `clearAllOaths` | `() => void` | activeOathIds = []. lockedAt 무시 |

**셀렉터·헬퍼**:

| 함수 | 반환 | 비고 |
|------|------|------|
| `isOathEnabledInField(fieldId, state)` | `boolean` | `FieldDef.boss != null && bossKillCounts[boss] > 0` (스펙 4-1) |
| `getEffectiveOathIds(state)` | `string[]` | `lockedAt != null`이면 snapshotIds, 아니면 빈 배열(효과 미적용) |
| `getEffectiveOathEffects(state, ctx)` | `OathEffect` (합성) | `ctx`로 적·필드 forbidOathIds 받아 차단된 맹세 제외 |
| `getEffectiveWeightSum(state, ctx)` | `number` | forbid 제외 후 weight 합 |

**호출부 연동** (필드 진입 액션):
1. `isOathEnabledInField(targetFieldId)` 체크
2. true → `lockOathsForField(targetFieldId)` 호출, lockedAt 박제
3. false → lockedAt = null 유지 (효과 자동 미적용, activeOathIds는 보존)
4. 필드 이탈 시: `unlockOaths()` (lockedAt = null)

**토글 게이트**: `state.currentField == null`일 때만 토글 허용 (마을에서만). 어떤 필드든 진입 후엔 거부.

**검증**:
- exclusiveGroup 라디오 동작 (같은 그룹 클릭 시 자동 해제)
- 마을 외부에서 토글 거부
- 미해금 필드 진입 시 lockedAt 박제 안 됨 + activeOathIds 보존
- 해금된 필드 이탈 시 자동 해제

### S3. 페널티 파이프라인 (S1·S2 의존)

각 페널티는 **그 효과를 호출하는 자리에서만** 적용. 절대 `GameState`를 직접 변형하지 말 것 (예: `maxHp`를 깎아두는 식 금지). **계산 시점에** `(1 - penalty)` 곱연산으로만 표현한다.

**모든 페널티 적용의 단일 게이트**: `state.oathSystem.lockedAt != null`. 이게 false면 즉시 단락 — 페널티·drain·보상 부스트 모두 적용 안 함. 미해금 필드/마을에선 자동으로 비활성화됨 (S2의 lockedAt 박제 룰로 자연스럽게 처리).

#### 3-1. 출력 데미지 페널티

- 위치: `damageCalc.ts`의 최종 산출 직전 (치명타·분산 다음 단계)
- 코드:
  ```ts
  const oathOut = effects.outDamagePenaltyPct ?? 0;
  damage = Math.floor(damage * (1 - oathOut));
  if (oathOut > 0 && damage < 1) damage = 1;  // 1 클램프 (-95% 단계 보호)
  ```
- 적용 대상: 절초·평타·반격·플레이어 장비 DoT (모든 출력 데미지)
- **율법/철칙과 곱연산 별도** — 적 받는 피해 감소(`guardDamageTakenMultiplier`, `bossDamageTakenMultiplier`)는 그대로, 그 위에 맹세 곱연산.

#### 3-2. 받는 피해 페널티

- 위치: `enemyCombat.ts`의 받는 피해 계산 최종 단계 (회피·블록·외법 감소 **이후**)
- 코드:
  ```ts
  const oathIn = effects.inDamageBonusPct ?? 0;
  incoming = Math.floor(incoming * (1 + oathIn));
  ```
- 적용 대상: 평타·스킬·fixedDamage·DoT 모두 포함

#### 3-3. 최대 내력 페널티

- 위치: maxQi (또는 maxStamina) 산정 자리. 외문수좌 `playerMaxQiBase`와 통일된 단일 원천을 사용할 것.
- 곱연산: `effectiveMax = baseMax * (1 - oathMaxQiPenalty)`
- 4단계(-100%): `effectiveMax = 0`. 내력 소모 무공 사용 차단 (이미 maxQi가 0이면 자연스럽게 막힐 가능성, 실제 동작 확인할 것)
- **변경하지 말 것**: 베이스 maxQi 자체. 페널티는 사용 시점 곱연산으로만.

#### 3-4. HP 회복 페널티

- 회복 함수의 단일 진입점 식별 → 거기서 mult 적용
- 자연 회복(gameLoop.ts hp regen) + 스킬 회복(배화교 묵념·심법 등 모든 heal 호출부) **모두 동일 곱연산** 적용
  ```ts
  const oathRecv = effects.hpRecoveryPenaltyPct ?? 0;
  healAmount = Math.floor(healAmount * (1 - oathRecv));
  ```
- 회복 호출부가 분산되어 있으면 **공통 헬퍼 `applyHealing(state, amount)`** 도입 후 그 안에서 처리할 것

#### 3-5. HP drain (회복 3단계)

- 위치: `gameLoop.ts` combat tick
- 조건: `currentEnemy != null` (전투 중에만)
- 매 tick:
  ```ts
  if (effects.hpDrainPctPerSec) {
    const drain = state.maxHp * effects.hpDrainPctPerSec * tickSeconds;
    state.hp -= drain;
    if (state.hp <= 0) /* 사망 처리 (기존 death flow 재사용) */;
  }
  ```
- **전투 외에는 정지** — 마을·답파 사이 정지 시 적용 안 함

### S4. 보상 부스트 (S1·S2 의존)

**위치**: `utils/combat/battleRewards.ts`

**전제 게이트**: `state.oathSystem.lockedAt != null` — false면 부스트 함수 자체를 호출하지 않음 (1.0 곱연산과 동일). 미해금 필드에선 자동 단락.

**구현**:
1. 헬퍼 함수 두 개 (스펙 5-3-1 의사코드 그대로):
   - `calcOathBoost(weightSum: number): { profMult: number; dropMult: number }` — piecewise + 캡 +1000%
   - `calcOathFlatBonuses(weightSum: number): { monsterRankBonus: number; extraDropTableUnlocked: boolean }`

2. **숙련도 분배**:
   - rank를 사용하는 자리에서 `effectiveRank = rank + monsterRankBonus`
   - 최종 gain에 `× profMult` 곱연산
   - rank 변경은 데이터 변형 금지, **계산 시점에만**

3. **드랍률 산정**:
   - 일반 드랍: `chance * dropMult`
   - `materials[id].excludeFromDropBonus === true`인 재료는 부스트 적용 X (그대로 베이스 확률)

4. **티어 2 추가 드랍 (확정 데이터, 카탈로그 5-3 참조)**:
   - 적 ID `baehwa_hwabosa` 처치 시 `extraDropTableUnlocked == true`이면 0.2% 확률로 `taoreuneun_bulggot_pyeon` 추가 롤
   - 적 ID `baehwa_gyeongbosa` 처치 시 동조건 0.4%
   - 두 재료 모두 `excludeFromDropBonus: true` — % 부스트 영향 없이 고정 확률
   - 데이터 위치: `data/oaths.ts`에 `OATH_TIER2_EXTRA_DROPS: Record<MonsterId, { materialId, chance }[]>` 또는 `monsters.ts`에 `tier2ExtraDrops?` 필드 — 작업자 판단

**검증**: 모든 가중치 구간(0/4/5/9/10/17/18/25)에서 부스트 수치가 카탈로그 5-1 표와 정확히 일치할 것.

### S5. UI — 맹세 탭 (S2 의존)

**컴포넌트**: `components/OathTab.tsx` 신규

**구성** (스펙 7번 + 카탈로그 0번 참조):

1. **상단 요약 패널**:
   - 활성 맹세 수 / 가중치 합 / 티어 (`적당한 도전`/`힘겨운`/`무모한`/`불가능한`)
   - 부스트 미리보기: `숙련도 +X% · 드랍률 +Y%`
   - 추가 보너스 표시 (등급 +1/+2, 일부 드랍 추가)

2. **카테고리 섹션 4개** — 카탈로그 1~4번 순서로
   - 각 카테고리 헤더에 카테고리명 + 현재 활성 단계 표시
   - 옵션 카드: 이름·한자·flavor·효과·weight·토글 버튼
   - 같은 `exclusiveGroup` 내 한 옵션 활성화 시 다른 옵션 비활성 표시 (라디오 동작 시각화)

3. **상태 배너**:
   - 마을 (`currentField == null`): 배너 없음, 토글 자유
   - 해금된 필드 진입 (`lockedAt != null`): "현재 [필드명]에서 잠김 — 이탈 시 변경 가능"
   - 미해금 필드 진입 (`currentField != null && lockedAt == null`): "이 필드는 [보스명] 처치 후 사용 가능 — 활성 맹세는 보존되며 해금된 필드에서 자동 발동"

4. **차단 표시**: `forbidOathIds`로 막힌 옵션은 회색 + 툴팁 "이 [필드/적]은 [맹세명] 무효"
   - 어느 forbid 컨텍스트를 보여줄지: **현재 진입 중인 필드**의 forbidOathIds + **답파 적·보스**의 forbidOathIds 합산
   - 해금 게이트(4-1)는 forbid와 별개 레이어 — 해금 게이트가 false인 필드는 옵션을 회색 처리하지 않고 위 3번 안내 배너로 충분히 표현

5. **스타일**: 기존 ArtsTab 카드 컨벤션 재사용 (CLAUDE.md "무공 카드 UI 규칙" 일부 차용 가능 — 카드형, 헤더 토글, 본문 펼치기). 단, 맹세 카드는 펼치기 없이 단일 헤더 + 효과 표시로 충분.

**라우팅**: `App.tsx` 탭 목록에 "맹세" 항목 추가. 위치는 무공·내공 다음, 인벤토리 앞 권장.

- 보상 부스트(숙련도·드랍률) 카탈로그 5-1 표와 일치
- 티어 3 등급 +1, 티어 4 +2 적용 여부
- 화보사·경보사 추가 드랍 발생 여부 (확률 통계)

**기존 balance-tester 패턴 따를 것** — `scripts/test-helpers.ts`·`testAdapter.ts` 활용.

## 3. 트랩·주의사항

| 트랩 | 어떻게 피하나 |
|------|---------------|
| 맹세 페널티를 GameState에 직접 박아둠 (예: `maxHp = base × 0.5`) | 절대 금지. 항상 사용 시점 곱연산으로만 표현 |
| 율법/철칙과 맹세를 합치거나 동일 식으로 처리 | 두 레이어를 명확히 분리한 곱연산 흐름 유지 |
| forbid 적용을 효과 단계에서만 처리 | 보상 산정의 weightSum에서도 동일하게 forbid 제외해야 함 |
| `excludeFromDropBonus` 재료에 % 부스트 곱함 | 곱연산 적용 전 재료 플래그 확인 |
| HP drain을 전투 외에서도 발동 | `currentEnemy != null` 가드 필수 |
| 회복 호출부 누락 (자연 회복만 처리, 스킬 회복 누락) | 단일 헬퍼 도입 + 모든 heal 호출부 grep 검수 |
| exclusiveGroup 라디오 동작 누락 | toggleOath에서 같은 그룹 자동 해제 로직 |
| 잠금 중 토글 허용 | toggleOath 가드 필수 |
| 외문수좌의 "3종 무공 장착 시 철칙 0%"를 맹세에 묶음 | 별개 메커니즘. 건드리지 말 것 |
| 미해금 필드에서 페널티·보상이 적용됨 | 모든 적용 자리에 `lockedAt != null` 단일 게이트. S2의 lockedAt 박제 룰을 정확히 지켜야 함 |
| 미해금 필드 진입 시 activeOathIds를 자동 해제 | Option B. 토글 보존, 효과만 미적용. 절대 해제하지 말 것 |
| 마을 외부에서 토글 허용 | toggleOath 가드: `state.currentField == null`만 통과 |
| 카탈로그 수치를 코드 상수로 하드코딩 후 카탈로그와 어긋남 | `data/oaths.ts`가 단일 원천 |
| 4단계 -95% 출력 페널티에서 데미지 0 발생 | 1 클램프 코드 누락 시 무력감 발생 — 적용 확인 |
