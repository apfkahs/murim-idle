# Part A: 심화학습 시스템

## A1. 개요

무공 등급 상승 시 **심화학습** 해금 → **무공 포인트 투자**로 활성화.
기존 등급 자동 패시브는 전부 심화학습 트리로 통합.
포인트 환불 가능.

---

## A2. 시스템 규칙

### 해금 조건

| 심화 단계 | 필요 성급 | 경지 제한 |
|-----------|----------|----------|
| 1단계 | 2성 | 없음 |
| 2단계 | 3성 | 없음 |
| 3단계 | 4성 | 삼류 중기 (tier ≥ 1) |
| 4단계 | 5성 | 삼류 후기 (tier ≥ 2) |

### 포인트 비용

| 심화 단계 | 비용 |
|-----------|-----|
| 1단계 | 1 |
| 2단계 | 1 |
| 3단계 | 2 |
| 4단계 | 2 |

### 핵심 규칙

1. **장착 중이어야 투자 가능**: equippedArts 또는 equippedSimbeop 상태일 때만 활성화/비활성화 가능.
2. **해제해도 심화 유지**: unequip 시 activeMasteries 데이터 보존. 재장착 시 효과 즉시 복구.
3. **효과 발동은 장착 시에만**: hasMastery()는 activeMasteries 등록 AND 장착 상태일 때만 true.
4. **전투 중 변경 불가**: battleMode !== 'none'이면 활성화/비활성화 차단.
5. **포인트 환불**: 비활성화 시 즉시 반환.
6. **포인트 풀 공유**: usedPoints = 무공 장착 cost + 심화학습 cost 합산.
7. **전체 심화 초기화**: 버튼 하나로 모든 무공의 활성 심화를 해제하고 포인트 전액 환불. 전투 중 불가. confirm 확인 필수.

### 투자 조건: 독립 vs 종속

심화학습은 기본적으로 **독립 투자** 가능 (성급+경지만 충족하면 순서 무관).
단, 일부 심화는 **종속 관계**가 있어 전제 심화가 활성화되어 있어야 투자 가능.

| 무공 | 종속 심화 | 전제 조건 |
|------|----------|----------|
| 삼재검법 | 삼재관통 (4단계) | 검기 잔류(1) 활성 필수. 이연격(2)/파쇄(3) 없으면 투자는 가능하나 효과 없음 (UI 경고 표시) |
| 삼재심법 | 심법 대성 (4단계) | 전투 수련(1) 활성 필수 |
| 흡공술 | 흡공 가속 (3단계) | 흡혈 강화(2) 활성 필수 |
| 무당보법 | 전부 독립 | — |

**종속 심화의 전제가 해제되면:**
전제 심화를 비활성화할 때, 종속 심화도 자동 비활성화 + 포인트 환불.

## A3. 무공별 심화학습

### 삼재검법 (액티브, cost 1)

| 단계 | 해금 | 이름 | 효과 | pt | 종속 |
|------|------|------|------|----|------|
| 1 | 2성 | **검기 잔류** | 미발동 시 데미지 = floor(현재 성급 power × 계수). 평타 5 대체 | 1 | 독립 |
| 2 | 3성 | **이연격** | 5% 2연타 (기존 double_strike) | 1 | 독립 |
| 3 | 4성 | **파쇄** | 3% 치명타 ×1.3 (기존 critical) | 2 | 독립 |
| 4 | 5성 | **삼재관통** | 검기 잔류 히트에도 이연격/파쇄 판정 적용 | 2 | 검기잔류 필수 (requires) |

**검기 잔류:** "현재 성급의 power" 기준으로 계산.

| 성급 | power | ×0.3 | ×0.35 | ×0.4 | 현재 평타 |
|------|-------|------|-------|------|----------|
| 2성 | 18 | 5 | 6 | 7 | 5 |
| 3성 | 26 | 7 | 9 | 10 | 5 |
| 4성 | 36 | 10 | 12 | 14 | 5 |
| 5성 | 48 | 14 | 16 | 19 | 5 |

> 계수는 balance-tester 결정. 방향: 0.3~0.4.

**미발동 시 우선순위:**
1. 무공 발동 성공 → 무공 데미지
2. 검기 잔류 활성 → floor(현재 성급 power × 계수)
3. 없음 → 평타 5

**삼재관통:** requires에는 검기잔류만 필수. 이연격/파쇄가 없으면 투자는 가능하지만 효과 없음. UI에 "이연격 또는 파쇄가 필요합니다" 경고 표시.

### 삼재심법 (심법, cost 0)

| 단계 | 해금 | 이름 | 효과 | pt | 종속 |
|------|------|------|------|----|------|
| 1 | 2성 | **전투 수련** | 전투 중 내공 = 비전투 생산 × combatRatio | 1 | 독립 |
| 2 | 3성 | **기혈순환** | 내공→HP 변환 추가 보너스 (기존 heal_bonus) | 1 | 독립 |
| 3 | 4성 | **내공 폭발** | 60초마다 burst (기존 neigong_burst). 비전투 시에만 | 2 | 독립 |
| 4 | 5성 | **심법 대성** | 전투 중 내공 회복률 추가 상승 | 2 | 전투수련 필수 |

> combatRatio 방향: 0.2~0.3.

### 어설픈 무당보법 (패시브, cost 1)

| 단계 | 해금 | 이름 | 효과 | pt | 종속 |
|------|------|------|------|----|------|
| 1 | 2성 | **경보** | hunt 킬 시에도 HP 5% 회복 (기존 post_battle_heal 확장) | 1 | 독립 |
| 2 | 3성 | **심득 강화** | 심득 +5% (기존 simdeuk_bonus) | 1 | 독립 |
| 3 | 4성 | **보법 숙련** | 회피 상한 25% → 30% | 2 | 독립 |
| 4 | 5성 | **무당 전신** | 회피 성공 시 HP 소량 회복 | 2 | 독립 |

### 조악한 흡공술 (심법, cost 0)

| 단계 | 해금 | 이름 | 효과 | pt | 종속 |
|------|------|------|------|----|------|
| 1 | 2성 | **전투 수련** | 전투 중 내공 생산 (삼재심법과 동일 구조) | 1 | 독립 |
| 2 | 3성 | **흡혈 강화** | 처치 시 HP 4% 흡수 (기존 lifesteal/lifesteal2 통합) | 1 | 독립 |

| 4 | 5성 | **사기충천** | 처치 시 다음 1회 공격에 처치한 적 심득의 n% 를 power에 가산. 중첩 불가 | 2 | 독립 |

> 사기충천 구체 수치는 balance-tester 결정.

## A6. 로직 변경 (combat-dev)

### hasMastery()

```typescript
function hasMastery(state: GameState, masteryId: string): boolean {
  for (const [artId, mIds] of Object.entries(state.activeMasteries)) {
    if (!mIds.includes(masteryId)) continue;
    if (state.equippedArts.includes(artId) || state.equippedSimbeop === artId)
      return true;
  }
  return false;
}
```

### executeAttack() — 검기 잔류

```
미발동 시:
  검기 잔류 활성 → damage = floor(현재 성급 power × RESIDUAL_RATIO)
  미활성 → damage = 5

이연격/파쇄 판정:
  if (fired || (isResidual && hasMastery('penetrate'))) { ... }
```

### tick() — 전투 중 내공

```
전투 중:
  ratio = 0
  if (hasMastery('simbeop_combat')) ratio = COMBAT_RATIO
  if (hasMastery('heupgong_combat')) ratio = HEUPGONG_RATIO
  if (hasMastery('simbeop_mastery')) ratio += BONUS
  if (ratio > 0) neigong += neigongRate * ratio * dt

burst: 비전투 시에만
```

### hunt 킬 시 경보

```
hunt 킬 + hasMastery('mudang_gyeongbo') → HP 5% 회복
```

### deactivateMastery — 종속 자동 해제

```
deactivateMastery(artId, masteryId):
  해당 심화 비활성화
  동일 무공 내 requires에 masteryId가 포함된 심화 → 자동 비활성화 + 포인트 환불
```

### getAllPassives() 제거

masteries + activeMasteries 기반으로 대체.

## E1. 개요

## E2. 시뮬레이션 방식

전투가 턴 단위 로직이므로 `dt = 경과시간` 한 번 처리는 불가.
**1초 단위 simulateTick() 반복**으로 정확히 시뮬레이션.

### tick() 리팩토링: simulateTick 순수 함수 분리

현재 tick()은 set() 콜백 안에서 상태를 직접 변형. 이를 순수 함수로 분리:

```typescript
// 순수 함수: 상태를 받아 새 상태를 반환 (set() 없음)
function simulateTick(
  state: GameState,
  dt: number,
  isSimulating: boolean
): GameState {
  // 현재 tick()의 로직을 그대로 수행하되:
  // isSimulating=true 시:
  //   - battleLog 누적 최소화 (마지막 10줄만)
  //   - 업적 체크 60틱마다만
  //   - floatingText 생성 중단
  // 새 GameState 객체를 반환
}

// 기존 tick()은 simulateTick을 래핑:
tick: (forceDt?: number) => {
  set(state => {
    const dt = ...; // 기존 dt 계산
    return simulateTick(state, dt, false);
  });
}
```

### processOfflineProgress

```typescript
processOfflineProgress: (elapsedSeconds: number) => OfflineResult;

function processOfflineProgress(elapsedSeconds: number): OfflineResult {
  let state = get();
  const maxTime = Math.min(elapsedSeconds, MAX_OFFLINE_SECONDS);
  let remaining = maxTime;

  // 통계 초기값
  const beforeNeigong = state.neigong;
  const beforeSimdeuk = state.totalSimdeuk;
  const beforeKills = state.totalYasanKills;
  let deathCount = 0;
  let battleTime = 0;
  let idleTime = 0;

  while (remaining > 0) {
    const wasBattling = state.battleMode !== 'none';
    state = simulateTick(state, 1, true);
    remaining -= 1;
    if (wasBattling) battleTime++; else idleTime++;
    if (state.battleResult?.type === 'death' && wasBattling) deathCount++;
  }

  set(state); // 한 번만 업데이트

  return {
    elapsedTime: maxTime,
    neigongGained: state.neigong - beforeNeigong,
    simdeukGained: state.totalSimdeuk - beforeSimdeuk,
    killCount: state.totalYasanKills - beforeKills,
    deathCount,
    battleTime,
    idleTime,
    achievementsEarned: /* 시뮬레이션 중 달성된 업적 ID */,
    dropsGained: /* 시뮬레이션 중 획득한 무공 ID */,
  };
}
```
