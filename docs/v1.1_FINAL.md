## 1. 전투 시스템 재설계

### 1.1 핵심 규칙

- 매 공격 시 장착 액티브 무공 중 랜덤 1개 선택 → 발동 판정.
- 실패 시 남은 후보 중 랜덤 1개 → 반복. 전부 실패 시 평타.
- 패시브 무공은 발동 판정 미참여. 장착만 하면 상시 효과.
- 심법은 심법 슬롯 별도. 전투 발동 무관.

### 1.2 공격 속도 (로그 곡선)

- **공식**: `4 / (1 + ln(1 + 경신 x 0.05))`
- **최소**: 1초 (회귀 없이). 회귀 후 0.5초 가능 (추후).

| 경신 | ln(1+경신x0.05) | 공격 간격 |
|------|-----------------|----------|
| 0 | 0 | 4.0초 |
| 10 | 0.41 | 2.8초 |
| 20 | 0.69 | 2.4초 |
| 30 | 0.92 | 2.1초 |
| 50 | 1.24 | 1.8초 |
| 80 | 1.61 | 1.5초 |
| 100 | 1.79 | 1.4초 |
| 150 | 2.08 | 1.3초 |
| 200 | 2.31 | 1.2초 |
| 300 | 2.71 | 1.1초 |
| 400+ | — | 1.0초 (캡) |

로그 곡선이라 초반(0→50)에 빠르게 줄고, 후반(100→200)은 완만. 플레이어와 적 각각 독립 공격 타이머. 적의 공격 간격은 몬스터 데이터에 고정.

### 1.3 타이머 처리

- **전투 시작 초기값**: `playerAttackTimer = 공격간격`, `enemyAttackTimer = enemy.attackInterval`. 양쪽 다 자기 간격만큼 기다린 후 첫 공격.

```
매 틱 (1초):
  playerAttackTimer -= 1
  enemyAttackTimer -= 1

  if (playerAttackTimer <= 0):
    플레이어 공격 처리 (1.4장)
    playerAttackTimer += 공격간격  // 잔여 이월

  if (enemy.hp <= 0): 처치, 적 공격 건너뜀, return

  if (enemyAttackTimer <= 0 && enemy.hp > 0):
    적 공격 처리 (1.8장)
    enemyAttackTimer += enemy.attackInterval  // 잔여 이월
```

플레이어가 먼저 판정. 내 공격으로 적이 죽으면 적 공격 없음. 타이머 잔여값은 이월: `+= 간격`으로 리셋하면 잔여 자동 보존.

### 1.4 무공 발동 로직

```
candidates = [...장착 액티브 무공]
fired = false

while (candidates.length > 0 && !fired):
  idx = randomInt(0, candidates.length-1)
  art = candidates.splice(idx, 1)[0]
  if (random() < art.triggerRate):
    dmg = art.power x 진영배율
    치명타/2연타 패시브 적용
    enemy.hp -= dmg
    로그: '{무공명}으로 {적명}에게 {dmg} 피해'
    fired = true

if (!fired):
  enemy.hp -= 5
  로그: '평타로 {적명}에게 5 피해'
```

- 매 공격마다 후보 풀을 새로 만들고 랜덤 순서로 시도.
- 같은 무공이 연속 턴에 나올 수도, 안 나올 수도 있음. 완전 랜덤.
- 무공 1개만 장착 시: 그 무공 발동 or 평타. 단순.
- 무공 3개 장착 시: 3번 기회. 평타 확률 낮지만 원하는 무공이 안 나올 수 있음.

### 1.7 경신 역할

- 경신 → 공격 간격만 (로그 곡선).
- 회피 기능 제거. 회피는 패시브 무공 전담.
- 기본 회피 0%. 무당보법 등 장착 시에만 발생.
- 회피율 상한 25%.

### 1.8 회피 판정

```
적 공격 시:
  totalDodge = 장착 패시브 무공 회피율 합산 (상한 25%)
  if (random() < totalDodge): 회피 (금색 로그)
  else: player.hp -= enemy.attackPower (흰색 로그)
```

### 1.9 전투 로그

공격 1회 = 로그 1줄. 공격 간격이 로그 속도 결정. 초반 4초에 1줄.

3색: 흰색(일반) / 금색(특수: 치명타,2연타,회피,드롭,보스등장) / 회색(시스템: 처치,등장,심득)


### 2.2 gameStore

```ts
totalSpentNeigong: number;

investStat(stat) {
  const cost = getStatCost(this.stats[stat]);
  if (this.neigong >= cost) {
    this.neigong -= cost;
    this.totalSpentNeigong += cost;
    this.stats[stat] += 1;
    this.maxHp = 50 + Math.floor(Math.log2(1 + this.totalSpentNeigong) * 15);
  }
}
```

```ts
if (state.totalSpentNeigong === undefined) {
```

## 10. 주의사항

- DPS 완전 폐기. UI에서도 제거.
- 공격 간격 공식: `4 / (1 + ln(1 + 경신 x 0.05))`. `Math.log`는 자연로그.
- 공격 간격 최소 1초: `max(계산값, 1.0)`.
- 타이머 잔여 이월: `+= 간격`. `-= 1` 아님에 주의.
- 전투 시작 시: `playerAttackTimer = 공격간격`, `enemyAttackTimer = enemy.attackInterval`.
- 플레이어 먼저 판정. 내 공격으로 적 사망 시 적 공격 없음.
- 경신 = 공격 간격만. 회피는 패시브 무공. 기본 회피 0%. 상한 25%.
- 무공 발동: 순서 아닌 랜덤. 매 공격마다 후보 풀 새로 생성.
- 패시브 무공: 발동 판정 미참여. 포인트만 소비.
- 평타 5. 추후 강화 가능.
- 몬스터 정보 숨김: killCounts 기반. 0/1/3/5/10/20회 단계.
- 지정 사냥: 이름 공개(1회+)된 몬스터만 선택 가능.
