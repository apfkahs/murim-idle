v8 주요 변경 (v7 기준)
  • 세로 HP 기둥 (아레나 레이아웃) — HP 바를 HUD의 가로 2줄에서 로그 양옆의 수직 기둥으로. 왼쪽 = 내 체력, 오른쪽 = 적 체력. 로그가 두 체력 기둥 사이 "전장"에 끼워지는 구조. 로그를 읽는 동안 시선 좌우 끝에 HP 상태가 상시 노출 → HUD로 올려보지 않아도 체력 확인 가능.
  • 바닥 앵커 · 위에서 아래로 깎임 — position: absolute; bottom: 0; height: N% 구조. 피해를 받을수록 위쪽 공간이 비워지는 "체력이 빠져나간다" 메타포. 25/50/75% 위치에 은은한 틱 마크로 잔량 판독 보조.
  • 포트레이트·이름·숫자 컬럼화 — 초상화(44x44), 이름, 현재/최대 체력 숫자, 상태칩, DPS를 양쪽 컬럼에 묶음. 상단 별도 HUD 블록 제거 → 수직 공간을 로그에 양보. 로그 영역이 v7 대비 세로로 약 130px 확보.
  • MP 보조 기둥 — 플레이어 HP 기둥 옆에 얇은 수직 MP 바(6px)를 나란히. 적은 HP만 단독 기둥(14px).
  • col-chip (컬럼 전용 칩) — 좁은 컬럼용 컴팩트 칩. 배화교 예시: 🔥 불씨 x3 / 흑풍채 보스 예시: ◇ 철벽 x5 -40%(2줄). 로그 본문의 .fire-chip / .shield-chip과 색·아이콘은 동일, 사이즈만 컴팩트. 위아래 시각 언어 일치 유지.
  • 기존 로그 규칙 그대로 — LATEST 스트립(◐ 최신), 2축 축선, 턴 단위 시간(24.0s), 위험 턴 배경, 대화·서술 좌우 정렬, Density 3단 토글 — v6~v7에서 확정된 규칙은 유지.
  • HP 델타 존속 — 기둥이 옆에 있어도 → 적 -18%는 유지. 이유: 로그를 스크롤해 과거 이벤트를 볼 때 그 공격 당시의 체감 %를 보존. 기둥은 "현재", 델타는 "그 순간".
기술적 타당성
  • 난도: 쉬움. CSS grid-template-columns: 64px 1fr 64px 3컬럼만 있으면 뼈대 완성. .vbar-fill은 bottom: 0; height: N% 절대 배치. 체력 깎이는 애니메이션은 transition: height 0.25s ease 한 줄.
  • 피해 체감 훅(선택) — 실제 구현 시 ".vbar-damage" 붉은 웨지 오버레이를 방금 깎인 구간에 0.4~0.8s 페이드 후 제거, 그 다음 bar-fill height 감소. 2단 애니메이션이 "맞음 → 깎임" 임팩트 전달에 효과적.
  • 좁은 화면 — 현재 각 컬럼 64px. < 360px 단말에서는 52px로 축소하거나 @media로 v7 가로 HUD 레이아웃 자동 전환.
  • 이름 길이 — 현재 "흑풍채 채주" 같은 긴 이름은 "채주"로 축약. text-overflow: ellipsis 적용. 필요 시 2줄 허용 룰 별도 결정.
  • 상태칩 3개 이상 — 버프/디버프가 많이 쌓이면 기둥이 세로로 길어짐. "+N" 접힘 규칙 필요(탭하면 팝오버로 전체 보기 등).
조정 가능 포인트 (피드백 바람)
  • 컬럼 폭: 현재 64px. 더 넓게(72px·80px) / 더 좁게(52px·56px)
  • 깎이는 방향: 현재 위→아래(위 공간이 비어감). 대안: 아래→위(모래시계형) / 중앙→바깥(v7의 가로 센터-앵커 방식 세로판)
  • 상태칩 위치: 현재 포트레이트 바로 아래. 대안: 바 옆쪽(바와 병렬) / 바 아래
  • HP 숫자 표기: 현재 1563 + /2349 2줄. 대안: 퍼센트만(67%) / 현재값만 / 안 보이게
  • MP 바: 현재 HP 옆 얇은 기둥. 대안: HP 안에 겹친 띠 / 숫자만 / 제거
  • HP 델타(→ 적 -18%): 기둥 있으니 삭제 가능. 유지 vs 제거 취향
  • 포트레이트 크기: 현재 44x44. 더 크게/더 작게
  • DPS: 현재 컬럼 맨 아래. 대안: 제거 / 포트레이트 아래로 이동 / 로그 헤더에 통합
