# 전투 로그 리팩터링 지시문

참조 목업: `combat-log-v6.html` — 이 파일을 실제 렌더링해보고 스타일·구조의 기준으로 삼을 것. 아래 명세는 v6를 모든 전투 로그에 일관되게 적용하기 위한 규칙서다.

---

## 1. 목표

현재 줄글 형태의 전투 로그를 **2축 타임라인 레이아웃**으로 전면 재구성한다. 플레이어가 "몇 초에, 누가, 뭘 했고, 얼마만큼의 피해/회복/상태이상이 발생했는지"를 스캔 한 번에 파악할 수 있어야 한다. 동시에 서술문(flavor)·대사(dialogue)의 문학적 맛은 보존한다.

## 2. 전체 컨테이너 구조

```
<log>
  <combat>                        // 전투 단위 — 적 하나당 하나
    <combat-header>               // 대상명·전투시간·피해/피격/디버프 총합·스파크라인
    <law-banner?>                 // 선언형 스킬이 발동되었을 때만
    <combat-body>
      <turn-group>*               // 타임스탬프 단위 이벤트 묶음
        <turn-label>              // 실제 경과 초 (예: 2.4s)
        <turn-events>             // 2축 그리드
          <event outgoing>        // 내가 적에게 한 행동
          <event incoming>        // 적이 나에게 한 행동 또는 나에게 적용된 상태이상
          <flavor side-left|side-right|both>
          <dialogue side-left|side-right>
    <kill-banner?>                // 처치 시에만
```

## 3. 타임스탬프 (턴 라벨)

- **T1/T2/T3 형태를 전부 폐기**. 실제 경과 초를 찍는다.
- 각 캐릭터·적의 공격속도(공속)를 기준으로 계산:
  - 내 캐릭터 공격: `1.0 / attackSpeed` 초 간격 (예: 공속 1.2s → 1.2, 2.4, 3.6...)
  - 적 액션(공격·회복·특수기): 각자의 고유 공속
  - 여러 이벤트가 같은 초에 겹치면 하나의 `turn-group` 아래 묶는다
- 표기: `2.4s` — 숫자는 JetBrains Mono, 단위 `s`는 한 단계 작고 흐리게
- 전투 헤더에 공속 설정 명시: `내 공속 1.2s · 적 회복 2.4s` 형태로 부제 노출

## 4. 이벤트 2축 레이아웃

`turn-events`는 가운데 세로 축선(1px, `#282832`)을 기준으로 2열:
- **왼쪽 (outgoing)** — 내가 적에게 가한 행동. 콘텐츠는 축 쪽으로 **오른쪽 정렬**(`justify-content: flex-end`). 숫자가 축에 붙는다.
- **오른쪽 (incoming)** — 적이 나에게 한 행동, 또는 나에게 적용된 디버프. 콘텐츠는 축 쪽으로 **왼쪽 정렬**.

이 배치의 목적: 숫자가 중앙 축 양옆에 마주 찍혀서 "가한 피해 vs 받은 피해"의 교환이 수직 스캔 한 번에 보인다.

## 5. 이벤트 내부 구성

각 이벤트 한 줄은 다음 순서로 flex:

**outgoing 예시:**
```
[태그] [스킬명] [상태칩?] [숫자]
```

**incoming 예시:**
```
[숫자] [주체명] [디버프칩?] [태그]
```
(숫자가 축 쪽에 먼저, 태그가 바깥쪽에 — 좌우 미러)

### 숫자 4단 위계 (monochrome, 크기로만 구분)

| 등급 | 클래스 | 크기 | 용례 |
|---|---|---|---|
| 치명 | `.value.crit` | 26px / 800 | 크리티컬, 킬샷 |
| 피격 대량 | `.value.hit-heavy` | 24px / 800 | 내가 받은 큰 피해 (폭발 등) |
| 비기/특수 | `.value.special` | 20px / 700 | 고급 스킬 |
| 일반 | `.value.normal` | 14px / 600 / 회색 | 평타 |
| 회복 | `.value.heal` | 14px / 600 / 청록 | 회복 (+304) |
| 무효 | `.value.muted` | 13px / 500 / 흐림 | 차단·회피 시 "—" |

**색은 쓰지 않는다. 오직 크기·굵기로만 위계를 만든다.**

## 6. 태그 (앞쪽 카테고리 박스)

태그에만 색을 쓴다. 폰트: Pretendard / 11px / 700 / letter-spacing 0.2px / padding 3px 9px.

| 태그 | 클래스 | 배경색 | 용례 |
|---|---|---|---|
| 치명 | `.tag.crit` | `#8a3440` (적색) | 크리티컬 히트 |
| 비기 | `.tag.special` | `#4a3e84` (보라) | 비기·특수기 |
| 피격 | `.tag.hit` | `#8a5430` (주황) | 내가 받은 피해 |
| 적회복 | `.tag.heal` | `#2e6448` (녹색) | 적의 회복 |
| 차단 | `.tag.block` | `#3a3a46` (회색) | 내 공격이 막힘 |
| 회피 | `.tag.dodge` | `#3a3a46` (회색) | 내가 회피 |

## 7. 상태이상 칩 (디버프 표기)

### 7-1. 불씨 (`fire-chip`) — 피격과 함께 또는 단독으로 적용될 때
```html
<span class="fire-chip">불씨<span class="count">+1</span></span>
```
- 갈색 톤(`#241d14` bg, `#d8a878` text)
- 단독 적용 시: incoming 쪽에 status-row로 배치 (태그 없이 칩만)
- 피격 이벤트와 함께 적용될 때: 피격 이벤트 `.side` 내부에 숫자·이름·태그와 같이 놓는다
- **`[상태이상]` 같은 별도 태그를 붙이지 말 것**. 칩 자체가 디버프임을 전달한다.

### 7-2. 무뎌짐 (`status-chip`) — 율법 등 지속 상태의 내 공격 영향 표기
```html
<span class="status-chip">무뎌짐</span>
```
- 차가운 회보라 톤, ▽ 프리픽스
- 율법이 active인 구간의 내 공격 이벤트 안에 고정 표기 (outgoing `.side` 내부)
- 차단되든 관통되든 상태 자체는 표기. 결과(차단/치명)는 태그로 따로 읽힌다.
- **"삼행 율법" 같은 스킬 이름으로 칩을 만들지 말 것**. 상태 이름(`무뎌짐`) 하나로 충분.

### 7-3. 일반 원칙
- 모든 디버프는 **내 상태** → incoming 쪽에 표기
- 같은 턴에 여러 디버프가 쌓이면 칩을 나란히 나열
- 전투 헤더 통계에 디버프 총합 표기 (예: `불씨 +7`)

## 8. 서술문 (flavor) / 대사 (dialogue) 위치 규칙

**주어가 누구냐에 따라 좌우 정렬을 결정한다. 이것이 가장 중요한 일관성 규칙.**

| 주어 | 위치 | 예시 |
|---|---|---|
| 내 몸·내 행동 | `side-left` (왼쪽 정렬, 오른쪽 보더) | "당신의 칼이 살갗을 가른다" |
| 적·적의 몸·적의 대사 | `side-right` (오른쪽 정렬, 왼쪽 보더) | "행자가 미소짓는다", "「아타르여...」" |
| 중립적 환경 묘사 | `both` (중앙) | "바람이 거세진다" (드물게) |

- **flavor**: 이탤릭, 11.5px, 회색. 보더 없음.
- **dialogue**: 이탤릭, 12.5px, 좀 더 밝은 회색. 2px 세로 보더로 구분.
- **minor / major 구분**: 밀도 모드 토글 시 `minor`는 compact에서 숨김, major만 노출. 극초반 조우·킬샷 직전 같은 큰 순간은 major.

## 9. 선언 배너 (`law-banner`)

율법처럼 "시작 시 선언되는 스킬"이 발동하면 `combat-body` 위에 full-width 중앙 배너로 분리한다.

```
<law-banner>
  <law-flavor>           // 셋업 묘사 (선택): "행자가 성화를 향해 두 손을 모은다."
  <law-name>             // 스킬명 (골드, letter-spacing 2px): "◈ 삼행(三行)의 율법 · 발동"
  <law-text>             // 선언 대사: "「삼행이 흐르지...」"
```
- 골드 상단 2px 라이트 바 + 다크 워밍 그라디언트 배경
- 이 블록 자체가 "특수한 뭔가가 발동했다"는 시그널 — 중복해서 칩을 달지 않는다
- 단, 영향 지속 구간의 내 공격에는 **무뎌짐 칩**을 개별 표기 (§7-2 참조)

## 10. 위험 턴 (`turn-group.danger`)

이 턴에 내가 받은 피해가 크거나 주요 사건이 발생하면 워밍 그라디언트 배경 + 턴 라벨 `▲` 마커 + 좌측 보더를 2px 오렌지로. 기준: **받은 피해가 한 턴 총 피해의 50% 이상이거나, 단일 피격 1000+**.

해당 턴의 주요 피격 이벤트에는 `hit-heavy-row` 클래스도 함께 적용 — 이벤트 자체에 warm 그라디언트 배경·3px 오렌지 좌측 바로 추가 강조.

## 11. 전투 헤더 (`combat-header`)

```
[대상명 · 전투시간(초)]  [공속 설정]           [피해 2,298] [피격 1,243] [불씨 +7] [턴별 스파크라인]
```
- **대상명**: 13px 볼드. 전투시간은 `· 7.2초 전투` 서브.
- **공속**: `내 공속 1.2s · 적 회복 2.4s` — 10.5px 흐린 회색. 초 수치는 Mono.
- **스탯**: 숫자는 Mono / tabular-nums / 볼드.
- **스파크라인**: Unicode block 문자 `▁▂▃▄▅▆▇█`로 턴별 피해 막대. 라벨 `1.2s 간격 피해` (또는 해당 공속). 1자리당 내 공격 1회 간격.

## 12. 처치 배너 (`kill-banner`)

```
★ 처치 · 배화교 행자
[검법 +21.0] [보법 +21.0] [권법 +21.0] [심법 +31.5]
```
- 상단 라이트 스윕, 골드 `★` 프리픽스
- 타이틀: Pretendard 13px 800 / letter-spacing 0.8px (빡빡하지 않게)
- 보상 칩: `[라벨(한글 Pretendard 600) + 값(Mono 700 tabular-nums)]` 구조로 분리. Mono 한 종류로 한글까지 찍지 말 것.

## 13. 밀도 토글 (3단계)

상단 controls에 `Full / Compact / Numbers Only` 3개 버튼 고정.
- **Full**: 모든 flavor·dialogue 노출
- **Compact (기본)**: `.minor` flavor·dialogue 숨김, `.major`만 노출
- **Numbers Only**: flavor·dialogue 전부 숨김, 숫자·태그·칩만

JS는 body에 `mode-full / mode-compact / mode-minimal` 클래스만 토글. CSS에서 selector로 분기.

## 14. 폰트 스택

```css
body: 'Pretendard', -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Malgun Gothic', 'Apple SD Gothic Neo', sans-serif;
숫자·타임스탬프: 'JetBrains Mono', 'SF Mono', monospace + tabular-nums
```
- 태그·상태칩·처치 타이틀 모두 한글 본문 폰트(Pretendard) 사용. 한글에 Mono 먹이지 말 것.
- 숫자는 무조건 Mono + tabular-nums (수직 정렬).

## 15. 색 팔레트 (다크 테마)

```
배경         #0a0a0d
패널         #15151a
헤더/구분자  #101014
보더         #26262c / #3a3a48
본문 텍스트  #e4e4e6 / #dadade
흐린 텍스트  #787880 / #6a6a72
숫자 강조    #f4f4f6
```

절대 규칙: **본문·숫자·서술문에 색을 쓰지 않는다.** 색은 태그·상태칩·선언 배너·danger 배경·처치 배너 ★ 에만 허용.

## 16. 반응형 (< 640px)

- 2축 그리드를 1열로 접기: outgoing·incoming 모두 왼쪽 정렬
- 가운데 축선 숨김
- flavor·dialogue 좌우 정렬 풀고 전부 왼쪽 정렬 + 왼쪽 보더

## 17. 체크리스트 (구현 후 확인)

- [ ] 모든 전투에서 T1/T2 같은 턴 번호가 사라지고 실제 초가 찍혔는가
- [ ] 공속 설정이 전투 헤더에 노출되는가
- [ ] outgoing은 중앙 축 왼쪽, incoming은 오른쪽에 일관되게 배치되는가
- [ ] 숫자는 크기로만 위계가 생기고 색이 들어가지 않았는가
- [ ] 태그에만 색이 들어가고 본문·이름·서술문은 monochrome인가
- [ ] flavor·dialogue가 주어(내 몸 vs 적)에 따라 올바른 쪽에 정렬되는가
- [ ] 디버프 적용에 `[상태이상]` 같은 중복 태그가 붙지 않았는가 (칩 단독)
- [ ] 율법 등 선언형 스킬은 중앙 배너로 분리되었는가
- [ ] 영향 지속 구간의 내 공격에 `무뎌짐` 칩이 고정 표기되는가 ("스킬 이름" 칩 아님)
- [ ] 큰 피격이 있는 턴에 danger 스타일과 hit-heavy가 적용되는가
- [ ] 서술문이 과도하게 압축되지 않고 원문 단위로 보존되었는가
- [ ] 밀도 토글 3단계가 동작하는가
- [ ] 처치 배너 보상 칩의 한글 라벨과 숫자가 분리된 폰트로 렌더되는가
- [ ] 모바일에서 2축이 1열로 접히는가

## 18. 작업 지침

1. `combat-log-v6.html`의 CSS를 **그대로** 프로젝트 스타일시트에 포함시키거나, CSS-in-JS/Tailwind로 1:1 포팅할 것.
2. 기존 로그 렌더러는 이벤트 스트림(시간, 주체, 액션, 값, 태그 목록, flavor 목록)을 받아서 위 구조로 변환하는 어댑터를 작성.
3. 공속·공격 간격은 전투 엔진의 실제 수치에서 직접 읽는다. 수동 하드코딩 금지.
4. 서술문 변형·대사는 기존 문학 자산을 존중한다. 압축·삭제 금지, 재배치만 허용.
5. 완료 후 대표 전투 3개(일반 전·회복형 적·선언형 스킬 보유 적)로 QA.

**최종 기준**: v6 목업을 시각적으로 100% 재현하면서, 이 규칙을 따라 어떤 전투 시퀀스든 동일한 톤과 구조로 자동 렌더링되어야 한다.
