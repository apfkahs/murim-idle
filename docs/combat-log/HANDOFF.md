## 배경

무림풍 방치형 게임 "무림방치록"(`D:\newidle`, Vite + React + TS + Zustand)의 전투 UI 리디자인.

**진행 경로:**
1. 전투 로그 가독성 리디자인 (v1 → v6, 완료·구현됨)
2. 실제 구현 후 스샷 확인 → HUD(위)와 로그(아래)가 시선상 분리되어 있는 문제 발견
3. HUD ↔ 로그 통합 목업 v7 제작 (현재 단계, 아직 미구현)

## 1단계 — v6 전투 로그 디자인 (구현 완료)

1. **2축 타임라인** — 중앙 세로 축선 기준. 좌측 = 내가 한 행동(outgoing), 우측 = 적이 한 행동 / 내가 받은 디버프(incoming)
2. **타임스탬프** — T1/T2 식 턴 번호 폐기. 실제 경과 초(`1.2s`, `2.4s` …) 표기. 전투 엔진의 실제 공속에서 계산
3. **숫자 4단 위계** — 색 없이 크기·굵기로만. 치명 26px / 피격대량 24px / 비기 20px / 일반 14px
4. **태그만 색상** — 치명(적), 비기(보라), 피격(주황), 적회복(녹), 차단·회피(회). 본문·숫자·서술문은 monochrome
5. **상태 칩** — 불씨(갈색, 🔥), 무뎌짐(회보라, ▽). `[상태이상]` 같은 중복 태그 금지, 칩 단독
6. **선언형 스킬** — 율법 등은 `law-banner` 중앙 블록으로 분리(골드 톤). 영향 구간 내 공격에는 `무뎌짐` 칩만 개별 부착
7. **서술문 좌우 정렬** — 주어가 내 몸이면 side-left, 적이면 side-right. 원문 압축 금지, 재배치만
8. **밀도 토글** — Full / Compact(기본) / Numbers Only 3단계
9. **헤더** — 대상명·전투시간·공속 설정·피해/피격/디버프 총합·턴별 스파크라인
10. **처치 배너** — ★ 프리픽스, 보상 칩은 "한글 라벨(Pretendard) + 숫자(Mono)" 분리
11. **위험 턴** — 한 턴에 큰 피격 있으면 warm 그라디언트 배경 + ▲ 마커 + hit-heavy 스타일

## 2단계 — v7~v10 HUD+로그 통합 탐색 (목업만, 미구현)

> v7~v10 모두 아직 실제 구현은 안 됨. 현재 프로덕션은 v6 로그 + 기존 HUD.
> v7부터 v10까지는 HUD 통합 방향성 탐색 목업. 유저 피드백: "끌리는 게 없다" — 일단 v10으로 쓰고 나중에 재검토.

### v7 — 중앙 축 연장 + HUD 가로 바 유지

실제 게임 스샷에서 발견된 문제: 위쪽 체력바·상태이상과 아래쪽 로그가 시각적으로 분리되어 시선이 위아래로 왔다갔다해야 함.

**v7 해결 방식 (A+B 하이브리드):**

1. **중앙 축선 연장** — HUD의 초상화 사이 세로선(`::after`)이 로그의 축선(`.turn-events::before`)과 픽셀 단위로 정렬. 위아래가 한 줄로 묶여 보이게
2. **칩 CSS 통일** — `.fire-chip / .status-chip / .shield-chip`을 HUD 상태 행과 로그 이벤트에서 동일 스타일로 사용. 위에서 본 불씨와 로그 안의 불씨가 같은 모양
3. **LATEST 스트립** — HUD 바로 밑에 고정된 "◐ 최신" 한 줄. 스크롤 중에도 최근 이벤트가 안 사라짐 (골드 톤 `#c0a060`)
4. **HP 델타 인라인** — 피해 숫자 옆에 `→ 적 -10%` 같은 상대 변화량 표시. 바를 안 봐도 임팩트 체감. 큰 피격은 `.hp-delta.heavy` 스타일(`#d29472`)
5. **바 앵커 반전** — 플레이어 HP는 중앙축 쪽으로 차오르고(`right-anchor`), 적 HP도 중앙축 쪽으로 차오름(`left-anchor`). 피해를 받을수록 바깥쪽으로 물러나서 가운데에 어두운 "전투선"이 생김
6. **보스 디버프(철벽) 표기** — `◇ 철벽 x5 -40%` 스택 수 + 효과치를 한 칩에 같이. 감소율 디버프도 같은 시스템으로
7. **ember 버그 대응** — 영구 디버프에 `Number.MAX_SAFE_INTEGER`(9007199254740991)가 초로 렌더링되는 버그. 영구 디버프는 duration 생략, 타이머형만 `(남은 X.Xs)` 표기

### v8 — 수직 HP 기둥 (아레나 레이아웃)

유저 제안: "체력 바를 세로로 쭉 길게 늘여서 전투로그 양옆에 배치, 맞을 때 아래로 깎이는 방식."

- HUD를 해체해 양쪽 컬럼(각 64px)에 세로로 재배치 (포트레이트 44x44 · 이름 · HP 숫자 · 상태칩 · 세로 HP 기둥 · MP 얇은 기둥 · DPS)
- HP 바는 `position: absolute; bottom: 0; height: N%` 바닥 앵커 → 맞으면 위쪽이 빈다
- 25/50/75% 위치에 틱 마크
- `col-chip` 컴팩트 칩 도입 (좁은 컬럼용)
- 중앙 로그 너비 ~268px (v7 대비 ~120px 축소)

**유저 피드백**: 포트레이트(44x44)랑 로그 영역이 너무 작아짐 → v9로 수정

### v9 — HUD 복원 + 얇은 세로 레일

- HUD를 v7 스타일로 원복 (포트레이트 64x64, 가로 HP/MP 바 부활)
- 세로 HP는 로그 양옆에 **20px 얇은 레일**만 남김. 순수 시각 바 (텍스트/포트레이트 없음)
- 레일에 25/50/75% 틱 마크 + "나"/"적" 미세 라벨
- HUD 가로 바와 세로 레일 이중 표현 (정밀 수치 ↔ 주변시 감지)
- 로그 영역 v7 수준으로 회복

### v10 — HUD 간소화 (현재 상태)

유저 지시: "세로 체력 바만 남기고, 위쪽엔 숫자랑 상태이상만 공간을 넓게 가져가서 쓰는 식으로. 위쪽 체력바는 지우고."

- **HUD 가로 HP/MP 바 전부 제거**. HP 시각화는 세로 레일 전담
- 현재 체력을 22px mono 800로 크게, 최대값·퍼센트(`67%`)는 작게
- MP는 HP 아래 서브로 14px 파란 계열
- HUD 내 상태칩은 13px로 크게 (padding 4px 12px, radius 5px). 로그 내 칩(10px)과 사이즈 위계 차등
- 세로 레일 20px → **24px**로 약간 두껍게 (HP 시각화 단일 채널이므로 존재감 강화)
- 포트레이트 64 → 68px
- 가로 점선으로 숫자 블록 ↔ 상태 블록 섹션 구획

**유저 피드백**: "끌리는 게 없네. 일단 지금 이대로 쓰고 나중에 수정하자" — v10을 현재 기준으로 유지, 추후 재탐색

## 이어서 할 작업 (택1)

**옵션 A — v7 HUD 통합 구현**
v7 목업대로 실제 HUD를 리팩토링. 주요 작업:
- 중앙 축선 연장 (HUD ::after ↔ 로그 ::before 정렬)
- 칩 컴포넌트 통합 (HUD 상태 행과 로그가 같은 `.fire-chip / .status-chip / .shield-chip` 사용)
- LATEST 스트립 컴포넌트 추가 (최근 이벤트 핀 고정, 자동스크롤 일시정지 훅)
- HP 델타 계산식: `damage / maxHP * 100`을 피해 라인마다 부가
- 바 앵커 방향 반전 (player → right-anchor, enemy → left-anchor)
- ember/dulled/shield 라벨 i18n 매핑 (`ember` → `불씨`, `dulled` → `무뎌짐`, `shield` → `철벽`)

**옵션 B — ember duration 버그 먼저 수정**
`Number.MAX_SAFE_INTEGER` 영구 디버프가 초 단위로 렌더되는 문제. 아마 `app/src/utils/combat/` 또는 `components/battle/` 쪽에서 duration 표시 조건 분기 필요.
- `duration === Infinity || duration >= 1e9` → duration 표기 생략
- 유한값만 `(남은 X.Xs)` 포맷

**옵션 C — v7 미세 조정**
v7 목업 자체에서 고칠 부분이 있으면 HTML/CSS 수정 후 다시 구현 단계로.
