v7 주요 변경 (v6 기준)
  • 중앙 축 연장 — HUD의 포트레이트/체력바/상태칩/DPS 모두 2축 그리드에 배치. HUD 중앙 축선과 로그 중앙 축선이 하나로 이어져 "위아래 한 몸"으로 인지됨. 체력바도 축 쪽으로 정렬: 내 체력은 오른쪽 앵커(축 쪽으로 붙음), 적 체력은 왼쪽 앵커(축 쪽으로 붙음) — 줄어들면 바가 바깥쪽(화면 가장자리)으로 후퇴하는 시각 메타포.
  • 상태칩 스타일 통일 — 상단 HUD의 🔥 불씨 x3, ◇ 철벽 x5 -40% 칩이 로그 안 칩과 동일한 CSS. 위에서 보이는 지속 상태가 아래 로그의 적용 시점과 시각적으로 같은 언어를 씀 → 시선이 끊기지 않음.
  • 최신 이벤트 고정 (LATEST 스트립) — HUD 바로 아래에 "◐ 최신" 라벨과 함께 가장 최근 이벤트 1개를 고정 슬롯으로. 골드 톤 미세 배경으로 나머지와 분리. 시선은 HUD ↔ 최신 좁은 영역에서 대부분 해결되고, 이력을 볼 때만 아래로 스크롤.
  • HP 델타 인라인 — 각 이벤트 숫자 뒤에 → 적 -10% / → 나 -5% 같은 상대 체력 변화율을 미세 텍스트로 병기. 체력바를 안 봐도 이 한 방의 임팩트가 즉시 읽힘 — 위아래 시선 이동의 핵심 원인(숫자 → 바 대조)을 제거.
  • 영구 지속 버그 수정 — 기존 ember x3 (9007199254740991s)(MAX_SAFE_INTEGER가 초로 찍히던 버그)를 🔥 불씨 x3로 교체. 영구 지속 디버프는 duration을 아예 표기하지 않는다. 시한부 디버프만 (남은 X.Xs)를 뒤에 붙일 것.
  • 수치 디버프 표기 (철벽 류) — ◇ 철벽 x5 -40% 패턴: 아이콘 + 이름 + 스택 + 효과율. 감소형 디버프는 푸른 회색(#8ab0d0), 점화형(불씨)은 갈색(#d8a878)으로 색상 구분. 로그 안의 공격 이벤트에도 철벽 -40% 칩을 붙여 "이 공격이 철벽의 영향을 받고 있음"을 표기.
  • LATEST 하이라이트 — LATEST 스트립 안의 턴 라벨은 골드 톤(#c8a868)으로, 다른 로그(#7a7a82)와 색 차이로 "방금"임을 보임. 동시에 hp-delta도 함께 찍혀서 최신 한 줄로 "무엇이 얼마만큼 깎였는지" 요약.
구현 체크리스트
  • [ ] HUD CSS를 로그 CSS와 같은 스타일시트에 두고 chip-base·fire-chip·shield-chip·status-chip 재사용
  • [ ] HUD 중앙 축선 ::after를 전체 container의 50% 위치에 수직 1px로 그려서 로그의 축선(.turn-events::before)과 픽셀 단위로 맞춤
  • [ ] 영구 디버프의 duration을 Infinity 또는 Number.MAX_SAFE_INTEGER로 관리 중이라면 UI 포맷팅 단계에서 유한하지 않은 값은 `(-)` 처리
  • [ ] HP 델타 계산: 적에게 가한 피해는 damage / enemyMaxHP, 받은 피해는 damage / playerMaxHP
  • [ ] LATEST 스트립은 최근 N초(예: 2초) 윈도우 버퍼. 새 이벤트 발생 시 기존 LATEST는 일반 로그로 이관
  • [ ] 로그 자동 스크롤: 사용자가 스크롤을 위로 올린 상태면 자동 스크롤 일시정지, "최신으로" 버튼 노출 (별도 기능)
  • [ ] 상태칩 라벨 i18n: 내부 키 ember → 표시 불씨, 내부 키 dulled → 표시 무뎌짐, 내부 키 shield → 표시 철벽
