## 바꿀 것

### A. 체력 / 내력 막대 — 좌우 완전 분리

**현재 문제**: 가로 막대 하나를 좌/우로 쪼개서 내 HP vs 적 HP 표시. 내력은 왼쪽 절반에만 있어서 오른쪽으로 뻗어 나감. 막대가 얇음.

**목표 구조**:
- 2열 그리드 (`1fr 1fr`). 가운데 침범 없음
- 각 열 내부에 **HP 바 (두껍게, ~20px)** → **MP 바 (~14px)** 세로 스택
- 막대 안에 `체력` `내력` 라벨 + `1,535 / 2,349` 같은 절대값 표기 (tabular-nums)
- **내 MP는 절대 적 영역으로 확장되지 않는다.** 적이 내력 없는 몬스터여도 내 MP 바는 자기 진영 폭만 차지
- 적이 내력을 가진 몬스터면 적 쪽 MP 슬롯에 채운다. 없으면 `내력 없음` 같은 비활성 placeholder
- 각 진영 아래에 **버프/디버프 칩 영역**. 불씨(ember) 같은 디버프는 여기에만 표시

**참고**: mockup의 `.bars` / `.side` / `.bar-wrap` / `.side-buffs` 구조.

---

### B. 전투 현황 카드 — 전면 개편

카드는 접기/펼치기 가능해야 함(chevron 클릭). 접힌 상태에선 헤더만 남고 `8.0s 진행중` 같은 요약만 노출.

카드 본문은 **위→아래 순서로 5 섹션**:

#### B-1. DPS 대결 게이지
- 좌: `내 DPS 52` (초록, 큰 숫자)
- 중: 좌우 분할 바. 내 DPS 비중 vs 적 DPS 비중
- 우: `적 DPS 64` (빨강, 큰 숫자)

#### B-2. 종합 판정 뱃지 + 격차 요약 (중요)
게이지 바로 아래 한 줄:
```
DPS −12 · HP −2,136        [⚠ 종합 열세]
```

**판정 로직 (매우 중요 — 반드시 지킬 것)**:
- DPS만으로 우열을 판단하지 않는다. **TTK(Time To Kill) 기반**으로 계산:
  - `내_TTK = 적_현재HP / 내_DPS`
  - `적_TTK = 내_현재HP / 적_DPS`
  - `ratio = 적_TTK / 내_TTK`
- 뱃지:
  - `ratio ≥ 1.3` → `✓ 우세` (초록)
  - `0.8 ≤ ratio < 1.3` → `— 호각` (회색)
  - `ratio < 0.8` → `⚠ 종합 열세` (빨강)
- **TTK 숫자값은 UI에 노출하지 않는다.** 판정 뱃지만 보여준다
- DPS 격차 / HP 격차 수치는 그대로 표기

#### B-3. 공격 타이밍 게이지
- `⚔ 내 공격 1.5s · 다음 0.9s` + 충전 바 (0 → 공속까지 실시간 차오름)
- `🛡 적 공격 2.5s · 다음 0.8s` + 충전 바 (오른쪽 정렬)
- 바 너비는 `(공속 경과시간 / 공속) × 100%`

#### B-4. 피해 카드 2장 (3장 아님)
- `누적 피해` — 큰 숫자(초록) + `평균 N/s · 최대 N`
- `누적 피격` — 큰 숫자(빨강) + `평균 N/s · 최대 N`
- **❌ 불씨 카드 / 디버프 카드를 추가로 만들지 않는다.** 디버프는 막대 아래 버프 칩에만 표시

#### B-5. 스킬 타임라인 (핵심 신규 기능)
나/적의 쿨다운 스킬 및 턴/초 기반 차징 스킬을 리스트로 표시.

**각 행의 컬럼**: `[진영] [아이콘] [스킬명] [상태 태그?] [진행 바] [남은 시간]`

**상태별 렌더링**:
| 상태 | 진행 바 | 태그 | 테두리 |
|---|---|---|---|
| 준비됨 | 100% 초록 | `준비됨` (초록) | 초록 강조 |
| 쿨다운 중 | `(쿨 경과 / 총 쿨)` 진영 색 | — | 기본 |
| 차징 / N턴 뒤 발동 | `(경과 / 발동까지)` 주황 | `⚠ 2턴 뒤 발동` (주황) | 주황 강조 |

- 남은 시간 표기는 **초 단위 / 턴 단위 자유**. 스킬 데이터의 단위에 맞춰 `"2.5s 후"` 또는 `"2턴 뒤 발동"`
- 차징 스킬은 발동 시점에 쿨다운 상태로 전환되는 라이프사이클을 따라야 함
- 내 게임에 스킬 / 무공 쿨다운 개념이 없다면 **이 섹션 자체를 생략**하고 계획서에 명시

### C. 로그 영역 상단에 탭 3개 추가

로그 영역은 디자인을 그대로 유지하되, 상단에 탭 바를 추가:

1. `전투 로그` (기본 활성, 기존 로그 그대로)
2. `내 정보`
3. `전투 통계`

탭 전환은 같은 영역 내에서. 로그창 높이를 바꾸지 않는다.

---

### D. 내 정보 탭 — 실제 게임 스탯만

아래 항목 중 **내 게임에 실제로 있는 것만** 렌더링:

**기본 스탯 섹션**
- 체력, 내력, 공격력, 방어력, 공격속도, 치명타율, 치명타 피해, 회피

**❌ 절대 추가하지 말 것 (내 게임에 없음)**
- 오행 상성 (火水木金土)
- 명중
- 관통 / 방어 관통
- 이동속도

**장착 무공 섹션**: 실제 장착된 무공 리스트. 슬롯 수는 게임 시스템에 맞춤. 빈 슬롯은 "비어 있음" 표시

**장착 장비 섹션**: 실제 장비창 기준 슬롯. 슬롯 개수 / 명칭 / 등급 체계는 게임 데이터에서 가져옴

---

### E. 전투 통계 탭

**현재 전투** (시작 시점 기준 누적):
- 누적 피해, 누적 피격, 순손익
- 치명타 횟수 / 최대 치명타 피해
- 회피 횟수 / 회피율
- 스킬 발동 횟수

**세션** (사냥 시작 이후):
- 처치 수, 경험치 획득
- 게임 재화 (내 게임의 실제 재화 이름으로 — `원보` 같은 placeholder 쓰지 말 것)
- 평균 DPS / 최대 DPS
- 생존율, 드롭 아이템 요약

데이터가 없는 항목은 렌더링하지 않는다.

## 일반 원칙

- **데이터가 없으면 렌더링도 없다.** 빈 슬롯 / 빈 섹션 대신 숨긴다 (또는 명시적으로 `없음` 표시)
- 숫자는 `font-variant-numeric: tabular-nums`로 자릿수 떨림 방지
- 색상 규약: 아군 초록 / 적 빨강 / 중립 회색 / 경고 주황 / 강조 골드. mockup의 CSS 변수 이름 참고
- 반응성: 기존 프로젝트의 반응형 규칙을 따른다 (독립적으로 추측해서 깨뜨리지 말 것)
- 애니메이션: 쿨다운/공속 바는 실시간 업데이트 (60fps 필수 아님, 100ms 간격 정도면 충분)
