# 무림 방치록 — 개발 가이드

이 문서는 게임의 설계 원칙, 밸런스 철학, UI 규칙, 테스트 방법을 정의한다.
구체적 수치는 포함하지 않는다. 수치는 코드와 데이터 파일이 진실의 원천이다.
이 문서는 "왜 그렇게 만들어야 하는가"를 설명한다.

> **충돌 처리 규칙**: 사용자의 지시가 이 가이드의 내용과 충돌할 경우,
> 가이드의 어떤 항목과 충돌하는지 구체적으로 설명하고,
> 해당 항목을 어떻게 수정할지 사용자에게 확인을 받은 뒤 진행한다.
> 확인 후 가이드를 사용자의 지시에 맞게 업데이트한다.

> **가이드 업데이트 규칙**: 새로운 시스템, 전투 방식, UI 변경 등
> 게임의 설계 원칙에 영향을 주는 변경이 이루어지면,
> 해당 내용을 이 가이드에 반영하여 항상 최신 상태를 유지한다.
> 가이드를 수정/추가한 경우, 작업 완료 보고 시
> 어떤 내용을 가이드의 어느 섹션에 어떻게 반영했는지 함께 보고한다.

> **작업 규모별 브랜치 규칙**:
> - **큰 작업** (UI 전면 개편, 시스템 재설계, 새 전장/전투 시스템 추가 등): git worktree로 별도 브랜치에서 작업한다. 완료 후 사용자 확인을 받은 뒤에만 메인에 합친다.
> - **작은 수정** (버그 수정, 수치 조정, 텍스트 변경, 색상 미세 조정 등): 메인 브랜치에서 직접 작업한다.

## 1. 게임 철학

### 핵심 루프
- 내공 수련(시간)으로 끊임없이 강해질 수 있다. 상한은 없다.
- 단, 전투를 병행하지 않으면 성장 효율이 급감한다.
- 전투에서 새 무공(특히 심법)을 얻으면 내공 생산이 점프하여 정체를 돌파한다.
- 이 사이클이 반복되다가 결국 벽에 부딪히고, 회귀로 넘어간다(추후).

### 재화 순환
- 내공(시간)과 심득(전투)은 서로를 필요로 한다.
- 내공만 쌓으면 스탯은 오르지만 무공이 약하다.
- 전투만 하면 무공은 강해지지만 스탯이 부족하다.
- 둘 다 해야 최적 효율이 나온다.

### 성장 감각
- 플레이어는 항상 "뭔가 오르고 있다"는 느낌을 받아야 한다.
- 멈춰있는 순간이 없어야 한다. 전투를 안 해도 내공은 쌓인다.
- 큰 성장은 드물게, 작은 성장은 항상.

## 2. 밸런스 원칙

### 난이도 곡선
- 같은 전장 내에서 약한 몬스터는 쉽게, 강한 몬스터는 성장 후에 사냥 가능해야 한다.
- 보스는 해당 전장의 모든 일반 몬스터를 안정적으로 사냥할 수 있는 시점에서도 바로 잡히지 않아야 한다.
- 보스를 잡으려면 추가적인 경맥 투자와 무공 성장이 필요하다.
- 경지 돌파는 의도적으로 오래 걸려야 한다. 급하게 달성되면 안 된다.

### 비용 곡선
- 경맥 투자 비용은 지수적으로 증가해야 한다.
- 새 심법 없이는 고레벨 경맥 투자가 비현실적으로 느려져야 한다.
- 새 심법을 얻으면 내공 생산이 뛰어올라 다시 투자가 원활해진다.
- 이 "정체 → 돌파" 사이클이 핵심 체감이다.

### HP와 생존
- HP는 투자한 내공 총량에 비례하되, 후반으로 갈수록 증가량이 체감 감소해야 한다.
- 어떤 스탯에 투자하든 HP가 올라야 한다. 특정 스탯을 HP 때문에 억지로 찍는 상황이 없어야 한다.
- 강체술 등 패시브 무공의 hpBonus로 추가 최대HP를 얻을 수 있다. 강체술 심화(gangche_reinforce)로 보너스를 1.5배 증폭 가능.
- 강한 몬스터와 싸우면 아파야 한다. 하지만 즉사는 안 된다(최소 몇 턴은 버텨야 함).
- 피해감소 메커니즘: 강체술 심화를 통해 5% 피해 감소(강인), HP 30% 이하 시 추가 10% 감소(불굴), 3% 완전 무효화(철벽지체)가 가능하다.
- 기본 회복은 전투 외에서만. 전투에 진입하기 전에 HP 관리를 하는 판단이 있어야 한다.
- 심화학습을 통해 전투 중 소량 회복(흡혈, 회피 시 회복, 처치 시 회복)이 가능하다. 단, 전투 중 회복량은 제한적이어야 한다.

### 무공 밸런스
- 강한 무공은 장착 비용(포인트)이 높아야 한다.
- 강한 무공은 성급 올리기도 어려워야 한다(필요 심득이 많다).
- 쓰레기 무공도 존재 가치가 있어야 한다. 발동 확률이 높아서 초반에 유용하거나, 패시브 효과가 독특하거나.
- 정파/사파 무공은 각각 강한 방식이 달라야 한다. 둘 다 최종 효율은 비슷하되 플레이 느낌이 다르다.
- 무공을 많이 장착하면 원하는 무공이 안 나올 수 있는 트레이드오프가 있어야 한다.

### 공격 속도
- 초반 공격은 느리다. 한 번의 공격이 묵직하게 느껴져야 한다.
- 성장할수록 빨라지되, 후반에 폭발적으로 빨라지면 안 된다. 로그 곡선으로 수렴.
- 공격 속도가 빨라져도 로그를 읽을 수 있는 속도여야 한다.

## 3. 전투 원칙

### 전투 흐름
- 매 공격 시 장착 액티브 무공 중 랜덤 하나를 발동 시도한다.
- 실패하면 남은 후보 중 다시 랜덤. 전부 실패하면 평타.
- 플레이어가 먼저 공격 판정. 적을 처치하면 적 공격은 발생하지 않는다.
- 공격 간격은 플레이어와 적이 독립적. 각자 자기 타이머를 가진다.

### 로그
- 공격 1회 = 로그 1줄. 로그가 곧 전투 체감이다.
- 로그 색은 3색 이하. 일반/특수/시스템.
- 같은 공격이라도 다양한 표현을 랜덤으로 보여준다. 반복감을 줄인다.
- 몬스터마다, 무공마다 고유 공격 표현이 있어야 한다.
- 로그가 너무 빠르면 읽을 수 없다. 초당 최대 2줄 이하가 이상적.

### 심화학습 (v2.0)
- 각 무공은 4단계 심화학습 트리를 가진다. 심화학습은 포인트를 소모하여 활성화한다.
- 심화학습은 성급 조건과 경지 조건을 충족해야 해금된다.
- 전제 심화가 있는 경우 해당 심화가 먼저 활성화되어 있어야 한다.
- 비활성화 시 종속된 심화도 함께 해제되며, 포인트는 즉시 환불된다.
- 전투 중에는 심화학습을 변경할 수 없다.
- 무공 미발동 시 검기 잔류로 평타를 대체할 수 있다 (삼재검법 심화). 검기 잔류 비율은 성급별로 차등 적용된다 (1~2성: 35%, 3성: 38%, 4성: 42%, 5성: 45%).
- 삼재관통 심화 시 검기 잔류에도 이연격/파쇄 판정이 적용된다.
- 사기충천(heupgong_morale): 적 처치 시 다음 1회 공격에 처치한 적의 심득 x 15%를 위력에 가산. 중첩 불가.
- 심화학습을 통해 전투 중 내공 생산, 회피 상한 상승, 흡혈 등 다양한 효과를 얻을 수 있다.

### 답파 vs 지정 사냥
- 답파는 고위험 고보상. 보스를 처치해야만 전체 보상을 받는다. 실패 시 전부 잃는다.
- 지정 사냥은 안전한 파밍. 매 처치마다 보상 확정.
- 두 모드의 역할이 명확히 다르다.

## 4. UI 원칙

### 전체 톤
- 어둡되 답답하지 않은 배경.
- 모든 카드/패널은 반투명. 배경을 완전히 가리지 않는다.
- 여백이 넉넉해야 한다. 빡빡한 느낌 금지.
- 모바일 게임처럼 세련되고 현대적이어야 한다. 대시보드나 관리 화면 느낌이 아니다.

### 색상
- 사용하는 색은 극도로 절제한다.
- 각 색상은 고유한 의미 하나만 가진다. 같은 색을 다른 용도로 쓰지 않는다.
- 형광색, 원색, 네온 효과는 절대 사용하지 않는다.
- 금색은 특별한 것에만. 남발하면 가치가 떨어진다.

### 폰트
- 산세리프 하나로 통일. 세리프를 섞지 않는다.
- 가벼운 무게(weight)를 기본으로. 굵은 볼드는 수치 강조에만.
- 숫자는 tabular-nums로 정렬.

### 버튼
- 기본은 투명 배경에 얇은 테두리. 눈에 띄지 않아야 한다.
- 호버/탭 시에만 약간 밝아진다.
- 눈에 확 띄는 색상 버튼은 만들지 않는다.

### 정보 공개
- 모든 정보를 한번에 보여주지 않는다.
- 플레이어가 경험을 통해 정보를 발견하는 재미가 있어야 한다.
- 몬스터 정보는 처치 횟수에 따라 점진적으로 공개.
- 미공개 정보는 ???로 표시하여 호기심을 유발.

### 전투 화면
- 전투 화면의 주인공은 전투 로그다.
- 캐릭터와 적 이미지는 분위기를 잡아주는 보조 역할.
- 로그가 실시간으로 쌓이는 것 자체가 게임 플레이의 핵심 체감.

### 심화학습 패널 (v2.0)
- 각 무공의 ArtDetail 내부에 심화학습 패널을 표시한다.
- 4가지 상태: 활성(금색 + 해제 버튼), 해금됨 미활성(투자 버튼), 미해금-성급부족, 미해금-경지부족.
- 전제 조건 미충족 시 경고 텍스트를 표시한다.
- 미장착 무공의 활성 심화는 "효과 비활성 중 (미장착)" 회색 표시.
- 전투 중 모든 심화학습 버튼 disabled.

### 포인트 표시 (v2.0)
- 포인트는 장착 비용 + 심화학습 비용 합산. getUsedPoints()로 가져온다.
- 미장착 무공에 심화 포인트가 할당된 경우 "(미장착 심화 n pt 포함)" 안내.
- 포인트 표시 옆에 [초기화] 버튼 (활성 심화가 1개 이상일 때만 표시).

### 헤더 (v2.0)
- 배속 토글: [1x] / [2x] 전환. 전투 중에도 변경 가능.
- 저장 버튼: 클릭 시 3슬롯 저장 모달 표시.
- 초기화 버튼은 제거. 저장 슬롯 모달에서 새 게임으로 대체.

### 모달 (v2.0)
- 모달은 반투명 오버레이(popup-overlay) + 중앙 카드(popup-content).
- 저장 슬롯 모달: 3개 슬롯 표시. 현재 슬롯에 "현재" 뱃지. 빈 슬롯은 "새 게임".
- 오프라인 결과 모달: 부재 시간 + 내공/심득/처치/사망 표시.

## 5. 밸런스 테스트 방법

### 시뮬레이션
- 게임 틱을 빠르게 돌리는 시뮬레이션 함수를 만들어 자동 테스트한다.
- 시뮬레이션은 "합리적인 플레이어"를 가정: 내공이 있으면 가장 비용 효율 좋은 스탯에 투자, 사냥 가능한 가장 강한 몬스터와 전투.
- 마일스톤: 각 전장의 약한 몬스터 안정 사냥 / 강한 몬스터 안정 사냥 / 보스 첫 클리어 / 경지 돌파.
- 마일스톤 간 도달 시간이 점진적으로 늘어나야 한다. 급격히 늘거나 너무 쉬우면 안 된다.

### 자동 밸런스 조정
- 시뮬레이션으로 마일스톤 도달 시간을 측정한다.
- 목표 시간 대비 너무 빠르면 몬스터를 강화하거나 비용을 올린다.
- 목표 시간 대비 너무 느리면 반대로 조정한다.
- 한 번에 큰 폭으로 바꾸지 말고 작은 단위로 반복 조정한다.

### 체크리스트
- 맨손으로 수련장을 클리어할 수 있는가?
- 첫 전장의 가장 약한 몬스터를 초기 무공으로 안정 사냥 가능한가?
- 첫 전장의 가장 강한 몬스터가 적절히 어려운가? (즉사 안 하지만 아픈가?)
- 보스가 충분히 어려운가? (성장 없이는 절대 못 잡는가?)
- 새 심법 획득 후 성장 속도가 체감될 정도로 빨라지는가?
- 경지 돌파 조건이 너무 쉽거나 너무 어렵지 않은가?
- 전투 중 HP가 0이 되기 전에 최소 5턴 이상 전투가 이루어지는가?
- 로그가 읽을 수 있는 속도로 나오는가?

### 마일스톤 측정 항목

balance-tester는 진행도 테스트 시 아래 항목의 도달 시간을 측정하고 보고한다.
"도달"의 구체적 기준은 테스터가 정의하되, 보고서에 명시한다.

**전장별 공통 마일스톤:**
- 전장 진입: 해당 전장의 몹을 처음 처치
- 안정 사냥: 가장 약한 몹을 반복 사냥해도 사망하지 않는 시점
- 중반 진입: 전장 중간 난이도 몹을 안정 사냥
- 최강 도달: 전장 최강 일반 몹을 안정 사냥
- 보스 클리어: 해당 전장 답파 성공
- 경지 돌파: 다음 경지 조건 충족

**측정 시 포함해야 하는 정보:**
- 도달 시점의 플레이 시간 (분)
- 도달 시점의 빌드 상태 (스탯, 무공, 경지)
- 이전 마일스톤으로부터 걸린 시간
- 해당 구간에서 주로 사냥한 몹

**건강한 밸런스의 징후:**
- 마일스톤 간 간격이 점진적으로 늘어난다
- 효율 추구 플레이와 도전 플레이의 최종 격차가 과도하지 않다
- 최적 사냥 대상이 성장에 따라 상위 몹으로 이동한다
- 특정 구간에서 지나치게 오래 정체하지 않는다

이 징후는 절대적 기준이 아니라 방향이다.
실제 밸런스는 테스트 데이터를 보고 기획자가 판단한다.

## 6. 코드 수정 시 규칙

### 금지 사항
- DPS 개념을 사용하지 않는다. 이 게임에 DPS는 없다.
- 형광색, 원색, 네온 효과를 CSS에 넣지 않는다.
- 세리프 폰트를 혼용하지 않는다.
- 전투 중 스탯 투자, 무공 교체를 허용하지 않는다.
- 전투 중 내공 생산은 심화학습(전투 수련 계열)을 통해서만 가능하다. 기본적으로는 비전투 시에만 내공이 생산된다.
- 로그 색을 3색 이상 사용하지 않는다.

### 변경 시 주의
- 공식을 변경할 때는 반드시 시뮬레이션으로 밸런스 영향을 확인한다.
- 몬스터 수치를 변경하면 반드시 해당 전장의 전체 밸런스를 재점검한다.
- 새 무공을 추가하면 기존 무공과의 포인트 대비 효율을 비교한다.
- UI를 변경할 때는 이 가이드의 UI 원칙을 준수한다.

### 새 콘텐츠 추가 시
- 새 전장은 이전 전장의 마지막 몬스터보다 약한 몬스터부터 시작해야 한다 (완만한 연결).
- 새 무공은 기존 무공과 역할이 겹치지 않도록 차별화해야 한다.
- 새 경지 단계는 이전 단계의 돌파 시간보다 길어야 한다.
- 새 업적은 자연스러운 플레이 과정에서 달성 가능해야 한다. 억지 조건 금지.

### 에이전트 협업 규칙

에이전트는 서로를 직접 호출하지 않는다.
작업 완료 시 기획자에게 다음 단계를 안내하고,
기획자가 다음 에이전트에게 지시한다.

**안내 패턴:**
- combat-dev가 전투 로직 변경 후:
  → "balance-tester로 밸런스 재검증을 권장합니다"
  → "testAdapter를 갱신했습니다" (해당 시)

- content-writer가 새 콘텐츠 추가 후:
  → "balance-tester로 새 콘텐츠의 밸런스 검증을 권장합니다"

- balance-tester가 보고서 제출 후:
  → "기획자의 옵션 선택이 필요합니다"
  → "선택 후 combat-dev에게 로직 변경이 필요합니다" (해당 시)
