# 배화교 외법 가지 — 하위 보법류: 성화보법(聖火步法) 스펙

배화교 비급 트리 중 **외법 가지에 편입되는 보법류 무공**. 원래 v0.4 문서상 보법 가지에 들어갔을 설계가, 유저 결정에 따라 **외법 가지로 통합**되며 단출한 노드 구성을 갖는다.

> **진행 상태 (2026-04-21)**: 이름·수치·정합성 안 B·만렙 정책(30 제한 아님, 세부 TBD) 확정. 배타 장착 구현·공속 하한 동적화·카운터 확률 데이터화는 시스템 확장 필요 항목으로 남음.

## 0. 시스템 변경 메모: 보법 → 외법 통합

**유저 결정**: "보법은 굳이 따로 항목을 넣지 않고 외법에 통합해도 될 것 같음. 노드가 많아질 필요가 없어보임."

해석:
- v0.4 문서 §4-1 비급 트리 구조의 **4가지(검법/심법/보법/외법)는 그대로 유지**
- 다만 **배화교 측 보법류 설계는 외법 가지의 한 무공**으로 편입 (보법 가지는 기존 보법 무공 전용으로 축소 또는 별도 운영)
- 외법은 원래 "다중 무공을 포함하는 가지"로 설계되었으므로 편입에 구조적 충돌 없음
- 각 외법 내 무공은 **노드가 단출하게** 구성됨 (본 무공은 단일 노드 + 특성 레벨로 설계)

**v0.4 문서 업데이트 필요 (별도 작업)**:
- §4-7 "심법/보법/외법 노드 구성 (임시)" → 심법은 식화심법으로 정식화됨, 보법은 본 스펙에 따라 외법 편입 방식으로 정리
- §4-1 트리 구조 설명에 "외법 가지 내 개별 무공은 노드가 단출함" 명시 필요할 수 있음

## 1. 이름

**확정**: `성화보법(聖火步法)` — 루비: **"아타르(Atar)의 걸음"**

- 배화교 내부 서사: 성화를 모시는 신도들이 성화를 든 채 발걸음을 흩뜨리지 않기 위해 수련한 발법
- 외부자(플레이어) 시점: 그 걸음을 훔쳐 자기 몸의 중심을 불꽃처럼 잠시 부유시키는 이단의 이동법
- 식화심법(식화=성화의 숨) + 성화보법(성화를 좇는 걸음) 네이밍 페어 — 둘 다 아타르(불의 신격)에 뿌리를 둔 대칭 축
- 검법 `염화검법(焰火劍法)` 은 "焰火"(타오르는 불)를 쓰지만, 심법·보법은 "息火"·"聖火"로 **성격이 나뉨** — 검법은 공격적 불, 심법·보법은 예배의 불 (서사적 차별화)

## 3. 도감/서사 초안

> 배화교의 수행자들이 성화를 이고 산등성이를 달리던 순례의 걸음. 그들은 내디딘 발이 성화를 흔들지 않도록, 그러나 성화가 꺼지지 않게 빠르게 — 그 균형을 평생 수련한다.
>
> 이 걸음을 훔친 이단자의 발 아래에선, 성화는 더 이상 지켜야 할 것이 아니다. 몸을 태우며 달리고, 태우며 피하고, 태워서 되돌려준다. 배화교의 순례자가 보았다면 기가 막혔을 걸음이다.

**톤**: 배화교 내부 서사(성화 순례)와 외부자(플레이어)의 역이용 서사를 양 시점에서 배치. 검법·심법과 동일한 이중 시점 구조.

유저 확정 필요: 톤 OK / 조정 / 재작성.

## 4. 시그니처 기믹

### 4-1. 개요

- **장착만으로 공격 속도 -1초** (2.5초 → 1.5초 = 시스템 하한)
- **기본 회피 10%, 회피 성공 시 20% 확률로 1.4배 피해의 카운터 공격**
- 레벨업 시 회피율·카운터 확률·공속 하한이 순차 성장

### 4-2. 시스템적 포인트

1. **공속 하한 `ATK_SPEED_MIN = 1.5초` 오버라이드 필요**
   - 본 무공 10/20/30레벨 특성은 이 전역 하한을 **본 무공 한정으로** 1.4 → 1.3 → 1.2로 끌어내린다
   - 기존 balance.ts의 `ATK_SPEED_MIN`을 상수에서 **동적 산출**로 변경. `calcPlayerAttackInterval`에서 장착 무공의 `minAtkSpeedOverride`를 참조하는 로직 추가 필요
2. **회피 카운터 확률이 하드코딩 50%**
   - 현재 `combatCalc.ts:471`, `enemyCombat.ts:645/688/968/998`, `baehwagyo/geombosa.ts:237`, `baehwagyo/howi.ts:189` 등 여러 곳에 `Math.random() < 0.5`가 하드코딩됨
   - 이를 `masteryEffects.dodgeCounterChance` (기본 0.5) 참조로 변경해야 본 무공의 "20% → 30레벨 50%" 구조가 기능
   - 녹림보법은 현재 50% 고정 — 기존 로직 변경 시 녹림보법도 `dodgeCounterChance: 0.5`로 명시해 기존 값 유지 (후방호환)
3. **배타 장착**
   - `artType: 'passive'` + passive 슬롯이 N개인데 녹림보법도 passive. 동시 장착 가능/불가 여부는 현재 슬롯 규칙에 따름
   - "이 보법 착용 시 다른 보법 불가" 유저 지시 → **강한 배타 장착** 필요
   - 제안: `footworkExclusive: true` 플래그를 `ArtDef`에 추가, 동일 플래그 무공은 서로 장착 불가로 명시
   - 또는 `proficiencyType: 'footwork'` 를 키로 "footwork 타입 무공은 한 번에 하나만" 규칙 도입 → 더 깔끔

### 5-2. 정합성 안 B 확정 — 30Lv 특성 묶음

**확정 (2026-04-21)**: 1레벨→29레벨까지의 매 레벨 증분(회피 +0.5%p, 카운터 +1%p)만으로는 30레벨에서 회피 24.5% / 카운터 49%에 그침. 30레벨에서 목표 수치(회피 25% / 카운터 50%)에 정확히 도달시키기 위해 **30Lv 특성 슬롯에 다음 효과를 묶어 적용**:

```
30Lv 특성 (묶음):
  · 공속 하한 1.3 → 1.2
  · 회피율 +0.5%p  (24.5 → 25.0)
  · 카운터 확률 +1%p (49 → 50)
```

- 10/20Lv 특성은 기존대로 공속 하한 하향만 담당 (1.5→1.4 / 1.4→1.3)
- 30Lv 특성에서만 공속 하한 + 수치 보정 두 효과를 묶음
- 심법 개방(baseCombatQiRatio 특성) · 재의 빠름(절초 연동 특성) 구조와 동일한 10/20/30 특성 레벨 패턴 유지

## 6. 배타 장착 상세

유저 지시: **이 무공 장착 시 다른 보법 무공 장착 불가**.

현재 녹림보법 한 종뿐이지만, 향후 보법류가 추가될 가능성 고려해 구조적 배타를 설계.

### 6-1. 구현 후보

| 방식 | 설명 | 장점 | 단점 |
|---|---|---|---|
| **A. footworkType 배타** | `proficiencyType: 'footwork'` 무공은 동시에 2개 이상 장착 불가로 전역 규칙 추가 | 단순, footwork 카테고리 일관성 | 다른 proficiencyType은 왜 배타 아닌지 규칙 불균일 |
| **B. 배타 플래그** | `ArtDef` 에 `mutuallyExclusiveWith?: string[]` 또는 `exclusiveGroup?: 'footwork'` 필드 추가 | 유연함. 특정 조합만 골라 막을 수 있음 | 메타데이터 관리 필요 |
| C. 슬롯 분리 | passive 슬롯 중 footwork 전용 슬롯 1개 분리 | 장착 UI 명확 | 대규모 리팩 |

→ **제안: B (exclusiveGroup 필드)**. 녹림보법·배화교 신 보법 모두 `exclusiveGroup: 'footwork'` 지정 → 한 번에 하나만 장착 가능. UI는 슬롯 토글 방식.

유저 확정 필요.

### 6-2. 스왑 규칙

- 플레이어가 본 무공 장착 시도 → 기존 녹림보법 자동 해제 후 장착? 또는 해제 후 장착 가능 메시지?
- 제안: **자동 해제 + 확인 다이얼로그** (실수 방지). 구현 시 별도 UI 작업 필요

## 8. 구현 메모

### 8-1. 신규 시스템 요구 사항

1. **공속 하한 오버라이드 시스템**
   - `balance.ts`의 `ATK_SPEED_MIN = 1.5` 상수 → 동적 산출로 전환
   - 장착 무공들의 `minAtkSpeedOverride` 중 **최솟값**을 참조 (여러 무공이 하한을 내리는 경우 가장 낮은 값 채택)
   - `ArtDef.masteries[].effects.minAtkSpeedOverride?: number` 필드 추가 → 10/20/30 특성 레벨에서 `1.4 / 1.3 / 1.2` 설정
   - `calcPlayerAttackInterval` (`combatCalc.ts:301`) 의 `Math.max(..., B.ATK_SPEED_MIN)` 를 `Math.max(..., effectiveMinAtkSpeed)` 로 교체

2. **회피 카운터 확률 동적화**
   - 하드코딩 `Math.random() < 0.5` 를 `Math.random() < (masteryEffects.dodgeCounterChance ?? 0.5)` 로 일괄 교체
   - 교체 위치 (확인됨):
     - `utils/combat/tickContext.ts:471`
     - `utils/combat/enemyCombat.ts:645, 688, 968, 998`

     - `utils/combat/skillHandlers/baehwagyo/howi.ts:189`

   - 신 무공은 레벨별로 20% → 49%(+특성 1% = 50%)로 증가

3. **배타 장착 필드**

   - 장착 처리 로직(스토어 추정 위치: `store/slices/artsSlice.ts`)에서 동일 `exclusiveGroup` 값의 기존 장착 무공 자동 해제

### 8-3. 밸런스 테스트 고려

- 공속 1.2초는 기존 시스템에서 도달 불가능한 영역. `playerCombat.ts` 의 각종 타이머·DoT 상호작용이 1.2초에서 정상 동작하는지 회귀 테스트 필요
- 30Lv 달성 시 `DPS ≈ 1/1.2 × ATK`로 기본 공속 대비 약 **2.08배 상승** — 상위 무공 해금 전까지의 최상위 선택지로 강력
- 이는 유저 의도(배화교 대면 필수 장비)와 맞물림. 삼행의 율법·철칙 해제 + 공속 돌파 + 회피 25%/카운터 50%라는 **전 구간 강 옵션**
