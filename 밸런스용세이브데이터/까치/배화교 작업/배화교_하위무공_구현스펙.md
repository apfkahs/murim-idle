# 배화교 하위 무공 구현 스펙 (코드 작업자용)

본 문서는 **식화심법(息火心法)** + **성화보법(聖火步法)** 두 무공을 코드에 반영하기 위한 통합 스펙표다.
설계 논의/서사 초안은 `배화교_하위심법_스펙.md`, `배화교_하위보법_외법통합_스펙.md`에 별도 보관되어 있으며, 본 문서는 **구현에 필요한 수치·구조·시스템 변경만** 담는다.

- 기준일: 2026-04-21
- 대상 커밋 위치: `app/src/`
- TBD 항목은 **구현 기본값(default)** 까지 박아 놓았으므로 유저 재지시가 없으면 그대로 채택.

## 0. 선행 시스템 변경 (두 무공 공통)

본 스펙을 이행하려면 아래 **5개 시스템 변경이 먼저** 필요하다. 각 변경은 기존 무공(마령심법·녹림보법 등)의 동작을 **변형 없이** 그대로 유지해야 한다.

### 0-1. `faction` 확장

| 파일 | 현재 | 변경 |
|---|---|---|
| `types/index.ts` `ArtDef.faction` | `'neutral' \| 'evil'` | `'neutral' \| 'evil' \| 'baehwagyo'` |
| UI 필터/색상(있다면) | — | `baehwagyo`는 일단 `evil`과 동일 톤으로 처리 (별색은 추후) |

### 0-2. `ATK_SPEED_MIN` 동적 오버라이드

현재 `calcPlayerAttackInterval`(`utils/combatCalc.ts:301`)은 전역 상수 `B.ATK_SPEED_MIN = 1.5`로 하한을 건다. 성화보법은 장착 상태에서 이 하한을 1.4/1.3/1.2로 내려야 한다.

| 추가 필드 | 위치 | 타입 |
|---|---|---|
| `MasteryEffects.minAtkSpeedOverride?: number` | `types/index.ts` | `number` (예: 1.4) |

`calcPlayerAttackInterval` 변경:

```ts
// before
return Math.max(raw, B.ATK_SPEED_MIN);

// after (의사코드)
const override = gatherMasteryEffects(arts, player).minAtkSpeedOverride;
const floor = override !== undefined ? Math.min(override, B.ATK_SPEED_MIN) : B.ATK_SPEED_MIN;
return Math.max(raw, floor);
```

`gatherMasteryEffects` 집계 규칙: 장착된 모든 무공의 현재 활성 특성 중 `minAtkSpeedOverride` 값들의 **최솟값**을 채택(여러 무공이 하한을 내리면 가장 낮은 값). 미정의면 1.5 유지.

### 0-3. 회피 카운터 확률 데이터화

현재 `Math.random() < 0.5` 가 6개소에 하드코딩됨. 이를 `masteryEffects.dodgeCounterChance`(기본 0.5)로 치환한다.

| 치환 파일 | 라인 기준 |
|---|---|
| `utils/combat/tickContext.ts` | 471 |
| `utils/combat/enemyCombat.ts` | 645 / 688 / 968 / 998 |
| `utils/combat/skillHandlers/baehwagyo/geombosa.ts` | 237 |
| `utils/combat/skillHandlers/baehwagyo/howi.ts` | 189 |

후방호환: 녹림보법 심득(`nokrim_bobeop_move1`) `effects` 에 `dodgeCounterChance: 0.5` 명시 추가해 기존 50% 확률을 보존.

새 필드:
```ts
MasteryEffects.dodgeCounterChance?: number;   // 기본 0.5
```

`gatherMasteryEffects` 집계 규칙: **최댓값** 채택(여러 무공이 동시 확률 버프를 줄 경우 가장 높은 값. 현재는 성화보법 + 녹림보법 동시 장착 불가이므로 단일 값만 가능).

### 0-4. `exclusiveGroup` 배타 장착

| 추가 필드 | 위치 | 타입 |
|---|---|---|
| `ArtDef.exclusiveGroup?: string` | `types/index.ts` | `'footwork'` 등 |

장착 처리(`store/slices/artsSlice.ts` 추정):
- 새 무공 장착 시, 동일 `exclusiveGroup` 값의 기존 장착 무공이 있으면 **자동 해제 후 장착** (다이얼로그 생략 — 기본값)
- UI는 장착 버튼 옆에 `※ <기존 무공명> 해제됨` 한 줄 toast 표시(간단히)

지정할 무공:
- 녹림보법 `nokrim_bobeop`: `exclusiveGroup: 'footwork'`
- 성화보법 `baehwa_seonghwa_bobeop`: `exclusiveGroup: 'footwork'`

### 0-5. 불씨 소각 주기 훅

식화심법이 장착된 상태에서 25초(또는 재의 빠름 레벨에 따른 감소된 주기)마다 `consumeEmberStacks(dots, 1)` 를 호출하고 재의 묵념·재의 맹세 효과를 트리거하는 훅이 필요.

| 신규 파일 | 역할 |
|---|---|
| `utils/combat/baehwagyoEmberTick.ts` | 식화심법 장착 감지 → 주기 계산 → 소각 → 후속 효과 트리거 |

등록 위치:
- `gameLoop.ts` 의 전투 틱 진입 직후(PRE_ATTACK 단계)에 호출
- 주기 타이머는 `tickContext.playerState.baehwagyoEmberTimer`(신규 필드) 에 누적
- 전투 진입(`onBattleStart`) 시 타이머 **0으로 리셋**(기본값)
- 전투 이탈 시 타이머 상태 **보존 안 함** (다음 전투 시작 시 0부터)

### 1-2. 기운 생산 coeff 테이블 (심법 개방 노드 레벨 → coeff)

`calcQiPerSec` 에서 기존 `proficiencyCoefficientByGrade[star]` 대신 **심법 개방 노드 레벨**을 키로 조회.

공식: `coeff(Lv) = 0.0970 + (Lv - 1) × 0.00303` (1 ≤ Lv ≤ 30)

| Lv | coeff | Lv | coeff | Lv | coeff |
|---|---|---|---|---|---|
| 1  | 0.09700 | 11 | 0.12730 | 21 | 0.15760 |
| 2  | 0.10003 | 12 | 0.13033 | 22 | 0.16063 |
| 3  | 0.10306 | 13 | 0.13336 | 23 | 0.16366 |
| 4  | 0.10609 | 14 | 0.13639 | 24 | 0.16669 |
| 5  | 0.10912 | 15 | 0.13942 | 25 | 0.16972 |
| 6  | 0.11215 | 16 | 0.14245 | 26 | 0.17275 |
| 7  | 0.11518 | 17 | 0.14548 | 27 | 0.17578 |
| 8  | 0.11821 | 18 | 0.14851 | 28 | 0.17881 |
| 9  | 0.12124 | 19 | 0.15154 | 29 | 0.18184 |
| 10 | 0.12427 | 20 | 0.15457 | 30 | 0.18487 |

- baseQiPerSec = **2.0 전 레벨 고정**
- Lv 31~50 coeff 정책: **TBD → 기본 구현은 Lv 30 코드에서 멈추고, 31 이상 투자 자체를 막는다** (UI에서 회색 처리)
- 기존 `calcQiPerSec` 는 마령/삼재 심법은 그대로 `proficiencyCoefficientByGrade` 사용, 식화심법은 새 경로로 분기

#### 1-3-4. 재의 빠름 — 주기 감소 + 절초 연동 특성

**주기 감소 테이블** (심법 개방 1Lv 상태에서 기본 25초 → 레벨별 감소):

| 재의 빠름 Lv | 주기(초) | 특성 |
|---|---|---|
| 0 | 25 | — |
| 1 | 20 | — |
| 2 | 20 − 5/8 = 19.375 | — |
| 3 | 18.75 | — |
| 4 | 18.125 | — |
| 5 | 17.5 | — |
| 6 | 16.875 | — |
| 7 | 16.25 | — |
| 8 | 15.625 | — |
| 9 | 15.0 | — |
| **10** | 15.0 (변동 없음) | **절초 발동 시 25% 확률로 불씨 1스택 즉시 삭제** |
| 11 | 15 − 5/9 ≈ 14.444 | — |
| 12 | ≈ 13.889 | — |
| … | (레벨당 -5/9초) | — |
| 19 | 10.0 | — |
| **20** | 10.0 | **절초 삭제 발동 시 25% 확률로 2스택 삭제로 전환** |
| 21 | 10 − 5/9 ≈ 9.444 | — |
| … | (레벨당 -5/9초) | — |
| 29 | 5.0 | — |
| **30** | 5.0 | **절초 발동 시 100% 확률로 불씨 1스택 삭제 (10Lv 특성 상위호환)** |

주기 공식 (코드용):
```ts
function baehwaEmberIntervalSec(fastLv: number): number {
  if (fastLv <= 0) return 25;
  if (fastLv === 1) return 20;
  if (fastLv <= 9)  return 20 - (fastLv - 1) * (5 / 8);    // 5레벨당 감소 구간 1
  if (fastLv === 10) return 15;
  if (fastLv <= 19) return 15 - (fastLv - 10) * (5 / 9);   // 구간 2
  if (fastLv === 20) return 10;
  if (fastLv <= 29) return 10 - (fastLv - 20) * (5 / 9);   // 구간 3
  return 5;                                                 // 30 (만렙)
}
```

**절초 연동 특성**(30Lv 도달 시 10Lv 특성 자동 무시, 20Lv 특성은 30Lv와 조합 유지):

- 10Lv: 절초 발동 시점 즉시 25%로 불씨 1스택 삭제
- 20Lv: 위 삭제 발동이 일어난 경우 그 중 25% 확률로 2스택 삭제
- 30Lv: 삭제 발동 확률 100%로 치환 (10Lv 특성 대체). 20Lv 특성과 조합 → 100% 발동 + 그 중 25% 2스택

`masteryEffects`:
```ts
effects: {
  emberBurnIntervalSec: baehwaEmberIntervalSec(Lv),
  emberUltDeleteChance:   Lv >= 30 ? 1.0 : Lv >= 10 ? 0.25 : 0,
  emberUltDeleteDoubleChance: Lv >= 20 ? 0.25 : 0,
}
```

자원·만렙:
- Lv 1~30: 희미한 잔불 **전용** (상위 재화 투자 불가 — UI에서 강제)
- Lv 31 이상 슬롯은 존재하지 않음(만렙 30 고정, 30→45→50 확장 제외)

### 1-4. 훅 발동 흐름

1. 매 전투 tick에서 `baehwagyoEmberTick` 훅 호출
2. 식화심법 장착되어 있고 심법 개방 Lv ≥ 1 이면:
   - `playerState.baehwagyoEmberTimer += dtSec`
   - `interval = baehwaEmberIntervalSec(fastLv)` 조회
   - `if (timer >= interval)`:
     - `timer -= interval`
     - **소각 처리**: 플레이어 불씨 스택 확인 → `consumeEmberStacks(dots, 1)`. 불씨 0이면 **스킵**(옵션 A 확정), 타이머는 빼지 않고 다시 0으로(스킵 틱 이후 재시작)
     - 소각 성공 시점 스택 수를 읽어 재의 묵념·재의 맹세 효과 트리거
     - 로그: *"불씨 하나가 조용히 꺼진다."*
3. **절초 발동 경로**(`playerCombat.ts` ult 발동 위치)에도 훅 추가:
   - `emberUltDeleteChance > 0` 이고 랜덤 확률 통과 시:
     - `doubleTrigger = rand() < emberUltDeleteDoubleChance`
     - `consumeEmberStacks(dots, doubleTrigger ? 2 : 1)`
     - 로그: *"절초의 바람에 불씨가 끌려 꺼진다."*

**빈 틱 처리 규칙 (기본 A=스킵)**: 불씨가 0이면 그 틱은 발동 무효. 타이머는 그 시점에 0으로 리셋돼 다음 interval 대기.

### 2-3. 레벨별 효과 테이블

| Lv | 회피 | 카운터 확률 | 카운터 배율 | 공속 하한 | 공속 보너스 |
|---|---|---|---|---|---|
| 1 | 10.0% | 20% | 1.4배 | 1.5초 | -1.0초 |
| 2 | 10.5% | 21% | 1.4 | 1.5 | -1.0 |
| … | (Lv당 회피 +0.5%p, 카운터 +1%p) | | 1.4 | 1.5 | -1.0 |
| 9 | 14.0% | 28% | 1.4 | 1.5 | -1.0 |
| **10** | 14.5% | 29% | 1.4 | **1.4** ★ | -1.0 |
| 11~19 | 계속 증가 | | 1.4 | 1.4 | -1.0 |
| 19 | 19.0% | 38% | 1.4 | 1.4 | -1.0 |
| **20** | 19.5% | 39% | 1.4 | **1.3** ★ | -1.0 |
| 21~29 | 계속 증가 | | 1.4 | 1.3 | -1.0 |
| 29 | 24.0% | 48% | 1.4 | 1.3 | -1.0 |
| **30** | **25.0%** ★ | **50%** ★ | 1.4 | **1.2** ★ | -1.0 |

**공식(코드용)**:
```ts
function seonghwaEffects(lv: number) {
  // 기본 성장
  let dodge   = 0.10 + (lv - 1) * 0.005;
  let counter = 0.20 + (lv - 1) * 0.01;
  let floorAS = 1.5;

  // 특성 레벨 오버라이드 (10/20/30)
  if (lv >= 10) floorAS = 1.4;
  if (lv >= 20) floorAS = 1.3;
  if (lv >= 30) {
    floorAS = 1.2;
    dodge   += 0.005;     // 30Lv 묶음 보정 (24.5→25.0)
    counter += 0.01;      // 30Lv 묶음 보정 (49→50)
  }

  return {
    bonusDodge: dodge,
    dodgeCounterChance: counter,
    dodgeCounterMultiplier: 1.4,
    minAtkSpeedOverride: floorAS,
    bonusAtkSpeed: 1.0,                 // -1.0초
    dodgeCounterEnabled: true,
  };
}
```

### 2-4. `MasteryEffects` 필드 요구

| 필드 | 타입 | 용도 |
|---|---|---|
| `bonusDodge` | number | 회피율 (기존 필드, 녹림보법이 이미 사용) |
| `dodgeCounterChance` | number | 회피 시 카운터 확률 (§0-3 신규) |
| `dodgeCounterMultiplier` | number | 카운터 배율 (기존 필드) |
| `minAtkSpeedOverride` | number | 공속 하한 (§0-2 신규) |
| `bonusAtkSpeed` | number | 공속 감소 (기존 필드) |
| `dodgeCounterEnabled` | boolean | 카운터 기능 on (기존 필드) |

### 2-5. 장착 규칙

- `exclusiveGroup: 'footwork'`
- 녹림보법 장착 중에 성화보법 장착 시도 → 녹림보법 자동 해제 + toast
- 반대 방향도 동일
