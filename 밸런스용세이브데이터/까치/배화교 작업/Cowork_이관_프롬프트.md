# Cowork 이관 프롬프트 — 무림방치록 배화교 기획 이어가기

아래를 Cowork 새 세션에 그대로 붙여 넣으면 된다.

---

## 복사해서 붙여넣기

```
무림방치록 프로젝트의 배화교 전장 기획을 claude.ai에서 진행하다가 Cowork로 이관했어. 이어서 작업하고 싶어.

## 먼저 파악해야 할 것

프로젝트 폴더에서 다음을 확인해:

1. 배화교 기획 문서들 (최신본이 기준)
   - `배화교_적설계_v0_5.md` — 전체 적 라인업, 위계 구조, 서사 설정
   - `배화교_시스템_정리_메모_v0_4.md` — 스킬트리/자원/비급 시스템
   - `배화교_스킬트리_UI_구현_프롬프트.md` — UI 구현 완료분
   - 혹시 더 최신 버전이나 관련 문서가 있는지도 확인

2. 현재 코드 구조
   - `app/src/data/arts.ts` — 무공 정의 (ArtDef, MasteryDef)
   - `app/src/data/monsters.ts` — 몬스터 정의 (MonsterDef, BossSkillDef, BOSS_PATTERNS)
   - `app/src/data/materials.ts` — 재료/비급/레시피 (MaterialDef, BijupDef, ArtRecipeDef)
   - `app/src/data/fields.ts` — 전장 정의
   - `app/src/CLAUDE.md` — 프로젝트 규칙

3. 스킬트리 UI 구현 결과물 — 어느 브랜치/파일에 있는지

## claude.ai 세션에서 확정한 것 (배화교 행자)

claude.ai 세션에서 배화교 첫 몬스터 "배화교 행자"의 스펙을 완성했어. 결과물은 `배화교_행자_스펙.md`로 저장해둘 예정이야 (아직 저장 안 했으면 내가 곧 붙여넣을게).

요약:
- id: baehwa_haengja, HP 3800, 공격력 160, 공격간격 2.5초, 등급 10
- 드랍: 희미한 잔불 2.5%
- 패턴 1 (삼행의 율법): 배화교 무공 미장착 시 피해 -50%
- 패턴 2 (평타 + 성화 송가): 70% 평타 / 30% 송가(자힐 + 확률 불씨 부여)
- 패턴 3 (아타르로의 귀의): HP 30% 이하 시 자폭 시퀀스
- 불씨 디버프: 스택당 최종 피해 -5% / 공격속도 -5%, 답파 단위 유지

## claude.ai 세션의 맥락 (톤·스타일)

- 배화교 = 조로아스터교 기반 + 무협 마교 톤 + 엘든 링 미친 불 느낌의 광화(狂火) 컨셉
- 조로아스터교 실제 교리(아샤/드루즈, Humata/Hūxta/Huvarshta, Atar 등)를 무공·적 설정에 녹임
- 명명: 보스·히든 등 중요한 적은 조로아스터 원어를 한국어로 표기 + 루비 병기
- 무공 이름은 "진정한 이름은 불꽃 속에 감춰져 있다" 식으로 숨기고, 상위 단계 도달 시 공개하는 서사 구조

## 이어서 할 작업

배화교 외문의 나머지 몬스터들을 하나씩 설계할 거야. v0.5 문서 §2-1에 따르면 외문 일반 5종 + 보스 + 히든:

- 배화교 행자 ✅ (완료)
- 배화교 호위 (다음)
- 배화교 검보사
- 배화교 화보사
- 배화교 경보사
- 외문 보스: 배화교 외문수좌 (Hērbed)
- 외문 히든: 아후라 마즈다의 그림자

내가 한 마리씩 설계 방향을 지시할 거고, 네가 행자와 같은 깊이로 스펙·패턴·로그를 함께 잡아줘. 행자 방식을 참고해:
- 수치(HP/공격력/공격간격/드랍)는 내가 직접 지정
- 패턴·스킬은 내가 큰 방향을 주면 네가 구체화
- 로그 문구는 네가 초안을 만들고 내가 조정
- 내가 말한 것 외에 임의로 추가하지 말 것

## 먼저 해줄 일

위 문서들을 읽고, 현재까지의 배화교 기획 전체 맥락을 파악해줘. 파악이 끝나면 "준비 완료. 행자 스펙 확인했고, 다음 몬스터 설계 시작 가능" 같은 식으로 알려줘. 그럼 내가 다음 몬스터를 지정할게.
```

## 추가 팁

### Cowork 시작 전 준비
- 이번 세션에서 저장한 `배화교_행자_스펙.md`를 프로젝트 폴더의 적절한 위치(예: `docs/배화교/` 또는 기존 배화교 문서들과 같은 곳)에 넣어두기
- 없으면 Cowork에서 첫 턴에 위 프롬프트 뒤에 이어서 스펙 내용 붙여넣으면 됨

### Cowork가 claude.ai보다 유리한 점 (활용 포인트)
- 다른 몬스터 설계할 때 네가 이미 써둔 배화교 관련 메모를 **내가 알아서 찾아봄**
- 행자 스펙이 `arts.ts` 기존 데이터와 충돌 없는지 **내가 직접 검증 가능**
- 설계 끝나면 `배화교/외문/호위.md` 같은 식으로 **즉시 저장해서 누적**
- 행자·호위·검보사를 하나씩 쌓아 올린 뒤, **외문 종합 문서**로 합치는 것도 가능

### 첫 턴에 추가로 시키면 좋은 것
위 프롬프트에 이어서 이렇게 덧붙일 수 있어:

> 파악 끝나면, 현재 배화교 기획 상태를 한 눈에 볼 수 있게 현황 요약표를 만들어줘. 어느 단계(외문/내문/사원/심처/지하)의 어느 몬스터가 어디까지 설계됐고 뭐가 남았는지. 앞으로 작업 진도 추적용이야.

이러면 Cowork가 처음부터 전체 진척도를 정리해둬서 앞으로 긴 작업이 한결 편해져.

## 참고

프롬프트에서 claude.ai 라고 명시한 이유는 Cowork가 과거 대화 내용을 **직접은 못 봄**. 그래서 "여기서 한 얘기를 토대로" 같은 표현 대신 "저장된 문서를 읽어라"로 지시하는 게 더 확실해. 중요한 결정은 반드시 파일로 저장해두는 게 좋아.
