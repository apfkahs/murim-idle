# 배화교 검보사 — 구현 프롬프트

배화교 외문 세 번째 몬스터 **검보사(劍步士)**의 코드 구현용 지시서. 완성된 스펙은 `docs/까치/배화교 작업/배화교_검보사_스펙.md` 참조. 이 문서는 해당 스펙을 **코드 구조에 맞게 단계별 작업으로 분해한 것**이다.

> ⚠️ 작업 전 반드시 읽을 것
> - `app/CLAUDE.md` — 프로젝트 규칙, 에이전트별 작업 범위, bossPatternState 40 필드 트리거
> - `docs/까치/배화교 작업/배화교_검보사_스펙.md` — 이 문서의 모든 수치·로그·분포의 원천
> - `app/src/utils/combat/skillHandlers/baehwagyo/howi.ts` (있다면) — 호위 핸들러를 패턴 참조용으로 사용
> - `app/src/utils/combat/emberUtils.ts` — 불씨 유틸의 현재 모양

## 전체 개요

검보사는 **HP 구간별 3태세(방어/공격/명인)** + **내력 게이지(성화의 일격 충전)** + **그로기** + **스턴 면역(명인 태세 한정)** + **불씨 스택 소모 스킬** 을 가진 외문 난이도 정점 몬스터. 기존 호위보다 구조가 한 단계 깊어 새 스킬 타입 2개(불씨 점화, 성화의 일격)와 새 태세 전환 로직이 필요하다.

구현은 아래 3 Phase로 나눈다. **Phase 간 의존성이 있으므로 순서 준수**.

| Phase | 담당 에이전트 | 주요 산출물 |
|---|---|---|
| **Phase 1 — 데이터** | `content-writer` | `data/monsters.ts`에 MonsterDef/BOSS_PATTERNS 추가, `data/fields.ts` 외문 구성 갱신 (필요 시) |
| **Phase 2 — 전투 로직** | `combat-dev` | `skillHandlers/baehwagyo/geombosa.ts` 신규, `emberUtils.ts` 확장(소모 함수), `store/types.ts` bossPatternState 확장, `registry.ts` 등록 |
| **Phase 3 — UI** | `ui-dev` | 내력 게이지 표시 (산군 방식 참조), 성화의 일격 경고 UI, 그로기 표시 |

### 1-2. BOSS_PATTERNS 추가

검보사는 isBoss=false이지만, 복합 스킬 핸들러가 필요하므로 BOSS_PATTERNS 구조를 사용한다. 호위·행자 선례를 따른다.

등록할 스킬 정의(이름만, 핸들러는 Phase 2에서 연결):

1. `geombosa_three_law` — 삼행의 율법 (행자·호위와 동일 메커니즘, 별도 핸들러지만 로직 공유 가능)

3. `geombosa_attack_router` — 태세별 공격 분포로 라우팅하는 상위 스킬
4. `geombosa_swordsmanship` — 검술 (배율 1.0)
5. `geombosa_flame_sword` — 화염검 (배율 1.8, 불씨 +1 확정)
6. `geombosa_flame_sword_combo` — 화염검 연격 (1.2 × 2타, 각 60% 불씨)
7. `geombosa_ember_ignition` — 불씨 점화 (공격 태세 전용, 플레이어 불씨 2 소모, 배율 2.2 + 소모당 +60%)
8. `geombosa_sacred_flame_strike` — 성화의 일격 (명인 태세, 게이지 100 시, 배율 8.0 + 불씨 +3)
9. `geombosa_grogy` — 그로기 처리 (6초 공격 스킵)

각 스킬의 배율·불씨 부여량·회피 가능 여부·로그 풀은 스펙 문서 해당 섹션 복붙하면 된다.

## Phase 2 — 전투 로직 (combat-dev)

### 2-1. `emberUtils.ts` 확장

현재 `emberUtils.ts`는 **스택 전체 제거(`removeEmberStacks`)**만 지원. 검보사의 불씨 점화는 **N 스택만 소모**하는 동작이 필요하다. 아래 함수를 추가한다.

```ts
/**
 * playerDotStacks에서 ember 스택을 정확히 count만큼 소모해 새 배열 반환.
 * - 현재 스택 수가 count 미만이면: 스택 그대로(삭제 안 함) 반환하며 soak=0
 * - 현재 스택 수가 count 이상이면: count만큼 차감, 0이 되면 엔트리 제거
 * 반환: { dots: 새 배열, consumed: 실제로 소모된 스택 수 }
 */
export function consumeEmberStacks(
  dots: DotStackEntry[] | undefined | null,
  count: number,
): { dots: DotStackEntry[]; consumed: number };
```

- `consumed < count`일 때 발동측이 스킬을 cancel / fallback 하는 판단은 호출자(핸들러) 몫
- `noDecay` 등 기존 필드는 유지

### 2-2. 새 핸들러 파일 — `app/src/utils/combat/skillHandlers/baehwagyo/geombosa.ts`

아래 책임들을 가진 단일 파일. 호위(`howi.ts`) 구조 참고.

**Guard clause**: 모든 훅·핸들러 첫 줄에서 `if (monster.id !== 'baehwa_geombosa') return;`

**훅 (PRE_SKILL_LOOP)** — 공격 타이밍마다, 스킬 선택 *전*에 실행:

1. **삼행의 율법 초기화** — 전투 첫 틱에 `threeLawLogged=false` 세팅, 플레이어 장착 무공 검사해 받는 피해 감소 플래그 설정
2. **태세 전환 검사** — 현재 HP %와 저장된 stance를 비교해 66% / 33% 임계 돌파 확인
   - 돌파 시: stance 갱신, `nextAttackSkipped=true`, 전환 로그 push, 명인 전환이면 `grogyEndAt=null`, `seonghwaGauge=0`, `stunImmune=true`
3. **그로기 체크** — `grogyEndAt`이 현재 시각 이후면 공격 타이밍 전체 스킵 (return early)
4. **스킵 플래그 체크** — `nextAttackSkipped=true`이면 소모하고 공격 타이밍 스킵

**훅 (IN_ATTACK_RESOLVE)** — 공격이 실제로 발동된 직후, 게이지 증가:

1. 명인 태세이고 방금 발동한 스킬이 **일반 공격(검술/화염검/화염검 연격)**이면 `seonghwaGauge += 20`
2. 게이지가 80을 처음 넘은 순간 `seonghwaWarningLogged=true` 체크 후 경고 로그 push
3. 방금 발동한 스킬이 `geombosa_sacred_flame_strike`이면 `seonghwaGauge=0`, `grogyEndAt = now + 6000` 세팅, 그로기 진입 로그 push

**핸들러 — `geombosa_attack_router`** (상위 라우터):

- 현재 stance에 따라 아래 분포로 하위 스킬을 선택
- 방어: 검술 50 / 화염검 35 / 화염검 연격 15
- 공격: `seonghwaGauge`가 아니라 **플레이어 불씨 스택**을 확인해:
  - 스택 ≥ 2 → 검술 30 / 화염검 25 / 화염검 연격 25 / 불씨 점화 20
  - 스택 < 2 → 검술 50 / 화염검 25 / 화염검 연격 25 (불씨 점화 슬롯을 검술로 치환, 분포 재정규화 없이 단순 슬롯 대체이므로 `0.30 + 0.20 = 0.50`)
- 명인:
  - `seonghwaGauge ≥ 100` → 성화의 일격 강제 발동
  - 그 외 → 검술 40 / 화염검 35 / 화염검 연격 25

**핸들러 — 개별 스킬들**:

- `geombosa_swordsmanship`: 단순 평타. 피해 = `공격력 × 1.0 × 분산 × stanceOutMult`
- `geombosa_flame_sword`: 피해 = `공격력 × 1.8 × 분산 × stanceOutMult`. 명중 시 불씨 +1 확정. 회피 판정 통상.
- `geombosa_flame_sword_combo`: 2타. 각 타 피해 = `공격력 × 1.2 × 분산 × stanceOutMult`. 각 타 **독립 회피 판정**. 명중 시 **각각 60% 불씨 +1**.
- `geombosa_ember_ignition`:
  1. 플레이어 불씨 스택 ≥ 2 확인 (아니면 호출자 라우터가 선택하지 않음, 방어적 체크)
  2. `consumeEmberStacks(..., 2)` 호출, `consumed=2` 확인
  3. 피해 = `공격력 × 2.2 × (1 + 2×0.6) × 분산 × stanceOutMult`
  4. 회피 판정: 회피 성공 시 **스택 소모도 무효** (consume 전에 회피 체크 선행하거나, consume 후 회피면 되돌리기)
  5. 불씨 부여 없음
- `geombosa_sacred_flame_strike`:
  1. 피해 = `공격력 × 8.0 × 분산 × stanceOutMult (=1.3)`
  2. 회피 판정. 회피 성공 시 피해·불씨 모두 무효
  3. **그로기는 회피 여부와 무관하게 진입** (IN_ATTACK_RESOLVE 훅이 처리)
  4. 명중 시 **불씨 +3 확정** (회피 불가·저항 불가 — 기존 `ignoreEvasion: true` 또는 유사 플래그 활용)

**피해 배율 헬퍼**:

```ts
// stance는 bossPatternState.geombosa_stance: 'defense' | 'attack' | 'master'
function stanceOutMult(stance): number {
  return stance === 'defense' ? 1.0 : 1.3; // 공격·명인 = 1.3
}
function stanceInMult(stance): number {
  return stance === 'attack' ? 1.0 : 0.7; // 방어·명인 = 0.7
}
```

`stanceInMult`는 **검보사가 받는 피해에 곱해지는 배율**로, damageCalc 체인의 `dmgReduction` 경로에 wire. 삼행의 율법 감소(-50%)와 **분리 계산되어 최종 곱연산**이 되도록 주의.

### 2-3. `store/types.ts` — bossPatternState 확장

아래 필드를 추가. 필드 수 8개 추가 → 현재 총합이 40개를 넘는지 **반드시 확인**. 넘으면 CLAUDE.md 규칙에 따라 사용자에게 리팩터 권장 알림 보고.

```
// 검보사 전용
geombosa_stance: 'defense' | 'attack' | 'master' | undefined;
geombosa_seonghwaGauge: number;          // 0~100, 명인 태세에서만 증감
geombosa_grogyEndAt: number | null;      // epoch ms, null이면 그로기 아님
geombosa_stunImmune: boolean;            // 명인 태세 진입 시 true
geombosa_threeLawLogged: boolean;
geombosa_attackStanceLogged: boolean;    // 66% 전환 로그 1회 플래그
geombosa_masterStanceLogged: boolean;    // 33% 전환 로그 1회 플래그
geombosa_seonghwaWarningLogged: boolean; // 80% 경고 1회/사이클, 게이지 리셋 시 false로
```

**네이밍 주의**: 모두 `geombosa_` prefix로 통일해 CLAUDE.md의 **몬스터 전용 필드 20개 초과** 검사 대상에 올바르게 집계되도록 함.

### 2-4. `registry.ts` 등록

`app/src/utils/combat/skillHandlers/registry.ts`에 신규 핸들러들을 등록:

- 각 `geombosa_*` 스킬 id → 해당 핸들러
- PRE_SKILL_LOOP 훅: 검보사 훅 함수
- IN_ATTACK_RESOLVE 훅: 게이지 증가 + 성화 후처리 훅 함수

`enemyCombat.ts`는 건드리지 않는다.

### 2-5. 스턴 면역 연동

명인 태세 진입 시 `stunImmune=true`가 세팅된다. 기존 `bossChargeStunImmune` 유사 플래그 경로가 있다면 그 경로에 검보사 케이스도 연결. 없다면 스턴 적용 지점(`enemyCombat` 또는 `playerCombat`의 스턴 부여 경로)에서 `monster.id === 'baehwa_geombosa' && bossPatternState.geombosa_stunImmune` 가드 추가.

스턴 면역 로그는 `onStunAttempt` 훅에서 최초 1회만 출력 (bossPatternState에 별도 플래그 두기보단 임시로 `geombosa_stunImmuneLoggedOnce` 추가 가능 — 필드 수 예산 고려해 판단).

## Phase 3 — UI (ui-dev)

### 3-1. 내력 게이지 시각화

- 위치: 검보사의 HP 바 **아래**
- 색: 흰색~연옥색 그라데이션 (성화의 "흰 불꽃" 컨셉)
- 값: `bossPatternState.geombosa_seonghwaGauge / 100`
- 명인 태세에서만 표시. 방어·공격 태세에서는 숨김
- 산군 내력 게이지 UI와 동일 레이아웃 재사용 권장

### 3-2. 80% 경고 이펙트 (선택)

게이지 80 이상일 때 게이지 바 주위 백색 펄스 애니메이션. 필수는 아니며 시간 허락 시 추가.

### 3-3. 그로기 상태 표시

`geombosa_grogyEndAt`이 현재 시각 이후인 동안 몬스터 이미지/프레임에 반투명 오버레이 + "그로기" 배지. 플레이어에게 딜 윈도우임을 명확히 알림.

### 3-4. 태세 배지 (선택)

체력바 헤더에 "방어 / 공격 / 명인" 텍스트 배지. 태세 전환 시 페이드 인/아웃 애니메이션. 선택 사항.

## 검증 체크리스트

Phase 완료 후 수동 또는 balance-tester로 다음을 확인한다.

**데이터 정합성**
- [ ] `baehwa_geombosa` MonsterDef가 정상 로드되고 외문 전장에 출현
- [ ] 희미한 잔불 드랍률 5%, 검법 비급 드랍률 2% (비급 id 결정 후)

**태세 전환**
- [ ] HP 66% 최초 돌파 시 로그 1회 출력, 다음 공격 1회 스킵
- [ ] HP 33% 최초 돌파 시 로그 1회 출력, 다음 공격 1회 스킵, 게이지 0, 스턴 면역 on
- [ ] HP가 플레이어 디버프나 치유로 상위 구간 복귀해도 태세 유지
- [ ] 방어 태세에서 받는 피해 `×0.7`, 삼행 조건 겹치면 `×0.35` 최종 적용 확인

**스킬 분포 (각 태세 20회 이상 표본)**
- [ ] 방어: 검술 50 / 화염검 35 / 화염검 연격 15 ±5%p
- [ ] 공격(스택 ≥2): 30/25/25/20 ±5%p
- [ ] 공격(스택 <2): 불씨 점화가 발동하지 않고, 해당 20%는 검술로 나타나 50/25/25 ±5%p
- [ ] 명인 일반: 40/35/25 ±5%p

**불씨 점화**
- [ ] 발동 시 플레이어 불씨 정확히 2 감소
- [ ] 회피 성공 시 불씨 소모되지 않음 (0 감소)
- [ ] 피해 = `공격력 × 4.84 × 분산 × 1.3` 범위 내

**성화의 일격**
