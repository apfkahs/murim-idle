# 무림 방치록 — 스탯 시스템 리디자인 설계서 v3

## 1. 개요

### 변경 목적
기존 선기/경신/마기 3스탯 체계를 **심(心)·기(氣)·체(體)** 체계로 교체한다.
새 시스템은 세 스탯이 서로 다른 축에서 기여하면서 균형 투자가 최적해에 가깝도록 설계한다.

### 핵심 원칙
- **스탯이 바닥, 무공이 색칠**: 스탯은 기반 수치를 결정하고, 무공은 그 위에 배율/특수효과를 얹는다.
- **몰빵 비효율**: 수확 체감(diminishing returns) 곡선으로 한 스탯 몰빵이 불리하다.
- **무공에 의한 빌드 차이**: 무공이 치명타 확률, 회피, 데미지 감소 등을 제공하여 빌드 방향이 달라진다.
- **파라미터화**: 모든 공식의 상수를 `BALANCE_PARAMS` 객체로 분리하여 코드 수정 없이 밸런싱 가능.

### 기존 시스템 대비 주요 변경
- 선기/경신/마기 → 기/심/체
- 방어력/회피 스탯 제거 → 무공에서만 제공
- HP 일원화 (방어력 공식 제거)
- 공격 간격을 스탯에서 분리 (기본 2.5초, 추후 무공에서 조절)
- 진영 배율 제거, ATK × 무공배율로 단순화
- 절초 시스템 추가 (내력 게이지 + 자동 발동)
- 치확은 기본 10% + 무공, 치피만 스탯에서 제공
- 기존 무공 전부 삭제, 삼재검법/삼재심법만 새 공식으로 재설계
- "내공" → "자연의 기운"으로 명칭 변경
- 심화학습 → **초(招)**로 명칭 변경
- 마이그레이션 없음, 세이브 초기화

## 2. 새 스탯 정의

| | 기(氣) | 심(心) | 체(體) |
|---|---|---|---|
| 공격 | 공격력(ATK) | 치명타 피해 | — |
| 자원 | — | 내력 풀 | 내력 회복 |
| 생존 | — | — | HP |

- **기(氣)**: 모든 피해의 base. ATK가 높아야 치명타든 절초든 의미가 있다.
- **심(心)**: 치명타 위력, 절초를 얼마나 많이 쓸 수 있는가.
- **체(體)**: 얼마나 오래 버티는가, 절초가 얼마나 자주 도는가.

## 3. 전투 수치 공식

모든 공식은 `weight × stat / (stat + half)` 패턴.
- **weight**: 천장 (최대 도달 가능한 값)
- **half**: 반감점 (이 스탯일 때 천장의 50%)

### 3.1 공격력 (ATK)
```
ATK = ATK_G_W × G/(G + ATK_G_H)
    + ATK_M_W × M/(M + ATK_M_H)
    + ATK_T_W × T/(T + ATK_T_H)
```

### 3.2 치명타 피해 (CRIT_DMG)
```
CRIT_DMG = CRITD_BASE + CRITD_M_W × M/(M + CRITD_M_H)
```
치확은 스탯에서 오지 않음. 기본 10% + 무공에서 추가. 상한 75%.

### 3.3 HP
```
HP = HP_BASE + HP_T_W × T/(T + HP_T_H) + HP_G_W × G/(G + HP_G_H) + hpBonus
```
`hpBonus`: 현재 미사용 (항상 0). 추후 장비/무공에서 제공 예정인 예약 필드.

### 3.4 내력 풀 (STAMINA)
```
STAMINA = STAM_BASE + STAM_M_W × M/(M + STAM_M_H)
```

### 3.5 내력 회복 (STAMINA_REGEN)
```
STAMINA_REGEN = REGEN_BASE + REGEN_T_W × T/(T + REGEN_T_H) + 해금된 MasteryDef의 bonusRegenPerSec 합산
```

```typescript
  HP_BASE: 500,

  HP_G_W: 100,     HP_G_H: 100,

  // ── STAMINA ──

  // ── NORMAL ATTACK GROWTH ──
  NORMAL_GROWTH_RATE: 0.02,   // 초식 배율 심득 성장률

  // ── QI PRODUCTION ──
  BASE_QI_PER_SEC: 1,           // 시스템 레벨 기본 기운 생산 (/초)
  QI_GROWTH_RATE: 0.075,        // 심법 기운 생산 심득 성장률

  // ── COMBAT QI ──
  COMBAT_QI_BASE: 0.05,       // 전투 중 기운 생산 시작 비율
  COMBAT_QI_GROWTH_RATE: 0.006,  // 전투 중 기운 생산 심득 성장률
  COMBAT_QI_CAP: 0.25,        // 전투 중 기운 생산 비율 상한

  // ── KILL BONUS ──
  KILL_BONUS_RATIO: 0.20,     // 처치 시 전투시간 대비 기운 보너스 비율

  // ── TAESAN (태산압정) — 기본값 참조용 ──
  // 실제 적용은 MasteryDef.effects.ultChange.simBonusW/H에서 읽음.
  // 새 무공의 절초 심 보정이 필요하면 해당 MasteryDef에 직접 정의.
  TAESAN_SIM_W: 1.5,   // 심 보정 최대 추가 배율 (삼재검법 4초 기준)
  TAESAN_SIM_H: 120,   // 심 보정 반감점 (삼재검법 4초 기준)

  // ── STAT COST ──
  COST_BASE: 10,
  COST_RATE: 1.15,
} as const;
```

**파라미터 위치 기준:**
- `BALANCE_PARAMS`: 공식에 들어가는 수학적 상수 (성장률, 캡, 반감점 등)
- `ArtDef / MasteryDef`: 무공마다 다를 수 있는 값 (초(招) 단계별 효과값: +5% 치확, +1/초 회복 등)

절초 코스트/배율/쿨타임은 무공별로 다르므로 각 ArtDef에서 관리.

## 5. 전투 시스템

### 5.1 공격 속도
스탯으로 조절하지 않음. 기본 2.5초. 추후 무공에서 조절.

### 5.2 초식 (일반 공격)
매 공격 타이밍에 가중치 기반 랜덤 1개의 초식 발동. 내력 소모 없음.
```
주공: 가중치 2 / 보공: 가중치 1
초식 피해 = ATK × 무공의 초식 배율
무공 없으면: ATK × BARE_HAND_MULTIPLIER (0.5)
치명타 시: 피해 × (CRIT_DMG / 100)
```

**초식 배율 성장 공식:**
```
normalMultiplier = growth.baseNormalMultiplier + (growth.normalGrowthRate ?? NORMAL_GROWTH_RATE) × sqrt(해당 무공 심득)
```
단, `normalMultiplierCap`에 의해 상한 제한. 초(招) 해금(`normalMultiplierCapIncrease`)에 의해 상한이 1.3 → 1.8 → 2.3으로 상승.

각 무공의 성장 공식은 **해당 무공의 심득**을 입력으로 사용한다 (전체 심득 합산이 아님).

삼재검법 기준 캡 도달 심득:
- 캡 1.3 도달: 심득 ≈ 900
- 캡 1.8 도달: 심득 ≈ 3,000
- 캡 2.3 도달: 심득 ≈ 6,400

### 5.3 절초 (강공격)
내력 ≥ 절초 코스트이고 쿨타임이 돌아왔으면, 초식 대신 주공의 절초 자동 발동.
절초 코스트, 배율, 쿨타임은 무공별로 다름.

**쿨타임 0 무공:** 쿨타임이 0이면 내력만 있으면 매 공격 타이밍에 절초가 발동된다 (의도된 설계). 이 경우 내력 코스트가 빈도를 제한하는 유일한 밸브이므로, **무공 설계 가이드라인: 쿨타임 0이면 코스트를 내력 풀의 50% 이상으로 설정할 것.**

### 5.4 내력 게이지
```
매 초: 내력 += 유효 내력 회복 (스탯 기반 + 무공 보너스)
내력 상한: STAMINA
전투 종료 시: 내력 0으로 리셋
지정 사냥(hunt) 중: 몬스터 간 내력 유지
```

### 5.5 피격
방어력 없음. 적 공격은 HP에 직접 적용.
```
피해 = 적 공격력 × (1 - dmgReduction / 100)
```
**음수 처리:** `dmgReduction`이 음수일 경우 `(1 - 음수/100)` > 1이 되어 피해가 증가한다. 이는 의도된 디자인으로, 특정 무공의 디메리트로 활용 가능.

회피 판정: 무공에서 제공하는 회피 확률로 판정. 회피 시 피해 0. 상한 `DODGE_CAP` (100%).

### 5.6 무공 장착 구조
- **주공 1개**: 초식 + 절초. 메인 딜.
- **보공 여러 개**: 초식만. 고유 효과로 기여.
- **심법 1개**: 자연의 기운 생산 보너스 + 내력 회복 보너스.

## 7. 무공 재설계

### 7.1 무공 데이터 구조

```typescript
export interface MasteryDiscovery {
  type: 'simdeuk' | 'boss' | 'event';
  threshold?: number;           // simdeuk 타입일 때 발견 심득
  bossId?: string;              // boss 타입일 때 보스 ID
}

export interface MasteryDef {
  stage: number;                // 초(招) 단계 (1초, 2초, ...)
  id: string;
  name: string;
  description: string;
  requiredSimdeuk: number;      // 해금에 필요한 심득
  requiredTier: number;         // 해금에 필요한 경지 (0이면 제한 없음)
  pointCost: number;
  requires?: string[];          // 전제 초(招) ID
  discovery?: MasteryDiscovery; // 발견 조건

  // 초(招) 효과 — 해금 시 적용. 해금된 MasteryDef의 effects를 합산하여 총 보너스 산출.
  effects?: {
    unlockUlt?: boolean;                  // 절초 해금
    bonusCritRate?: number;               // 치명타 확률 가산
    bonusDodge?: number;                  // 회피 확률 가산 (%)
    bonusDmgReduction?: number;           // 데미지 감소율 가산 (%)
    bonusRegenPerSec?: number;            // 내력 회복 가산 (/초)
    bonusQiPerSec?: number;               // 기운 생산 가산 (/초)
    bonusCombatQiRatio?: number;          // 전투 중 기운 생산 비율 가산
    normalMultiplierCapIncrease?: number; // 초식 배율 상한 증가분
    ultChange?: {                         // 절초 변경
      name?: string;                      // 절초 이름 교체
      simBonusW?: number;                 // 심 보정 weight (TAESAN_SIM_W)
      simBonusH?: number;                 // 심 보정 half (TAESAN_SIM_H)
    };
    killBonusEnabled?: boolean;           // 처치 시 기운 보너스 활성화
    synergyArtId?: string;                // 시너지 대상 무공 ID (없으면 무조건 적용)
  };
}

export interface ArtGrowth {
  // 초식 배율 성장 (active 무공)
  baseNormalMultiplier?: number;    // 시작 배율 (예: 0.7)
  normalGrowthRate?: number;        // sqrt 성장률 (없으면 BALANCE_PARAMS.NORMAL_GROWTH_RATE)

  // 심법 기운 생산 성장 (simbeop)
  baseQiPerSec?: number;            // 시작 생산량 (예: 1.0)
  qiGrowthRate?: number;            // sqrt 성장률 (없으면 BALANCE_PARAMS.QI_GROWTH_RATE)
  maxQiPerSec?: number;             // 생산량 캡 (예: 3.0)

  // 전투 수련 성장 (2초 해금 후)
  baseCombatQiRatio?: number;       // 시작 비율 (없으면 BALANCE_PARAMS.COMBAT_QI_BASE)
  combatQiGrowthRate?: number;      // sqrt 성장률 (없으면 BALANCE_PARAMS.COMBAT_QI_GROWTH_RATE)
  maxCombatQiRatio?: number;        // 비율 캡 (없으면 BALANCE_PARAMS.COMBAT_QI_CAP)
}

export interface ArtDef {
  id: string;
  name: string;
  faction: 'neutral' | 'righteous' | 'evil';
  artType: 'active' | 'passive' | 'simbeop';
  cost: number;

  role: 'main' | 'sub' | 'simbeop';  // 주공 / 보공 / 심법

  // 초식 (심법은 생략)
  normalMultiplierCap?: number;   // 초식 배율 상한 (초(招)로 상한 상승 가능)
  normalMessages?: string[];

  // 절초 (초(招) 해금 후 사용 가능)
  ultMultiplier?: number;
  ultCost?: number;
  ultCooldown?: number;           // 쿨타임 (초). 0이면 내력만으로 제한 (의도된 설계).
  ultMessages?: string[];

  // 심득에 의한 연속 성장 파라미터
  growth: ArtGrowth;
  // 초(招) — 단계적 해금
  masteries: MasteryDef[];
}
```

**역할 분리:**
- `ArtGrowth`: 심득에 의한 **연속 성장** (초식 배율, 기운 생산, 전투 수련 비율). 각 필드 생략 시 `BALANCE_PARAMS`의 기본값 사용. 전투 수련 파라미터는 해당 무공의 2초(전투 수련)가 해금된 상태에서만 활성. 미해금 시 `combatQiRatio = 0`.
- `MasteryDef.effects`: 초(招) 해금에 의한 **단계적 보너스** (치확, 회피, 감소, 내력회복, 배율 상한 증가, 절초 변경 등). 해금된 MasteryDef의 effects만 순회하여 합산. `synergyArtId`가 있는 MasteryDef의 effects는 해당 무공이 장착 중일 때만 적용. 미장착이면 해당 effects 전체를 스킵.
- `ArtDef`: 무공의 정적 속성 (초식 상한, 절초 기본 배율/코스트/쿨타임, 메시지 등).

새 무공을 추가할 때 코드 변경 없이 데이터만 추가하면 된다.

### 7.2 삼재검법 (주공, 균형형)

**기본 정보:**
```
id: 'samjae_sword'
name: '삼재검법'
faction: 'neutral'
artType: 'active'
role: 'main'
cost: 1
```

**초식:**
```
normalMultiplierCap: 1.3         // 초기 상한 (초(招)로 상승)
normalMessages: ['삼재검법의 검기가 빛난다!', '삼재의 이치를 담은 일격!']

growth: {
  baseNormalMultiplier: 0.7      // 시작 배율
  // normalGrowthRate 생략 → BALANCE_PARAMS.NORMAL_GROWTH_RATE (0.02) 사용
}
```

**초식 배율 성장 구체 공식:**
```
normalMultiplier = growth.baseNormalMultiplier + (growth.normalGrowthRate ?? NORMAL_GROWTH_RATE) × sqrt(삼재검법 심득)
               = 0.7 + 0.02 × sqrt(삼재검법 심득)
```
단, `normalMultiplierCap`에 의해 상한 제한.
- 캡 1.3 도달: 심득 ≈ 900
- 캡 1.8 도달: 심득 ≈ 3,000
- 캡 2.3 도달: 심득 ≈ 6,400

**절초 (1초 해금 후):**
```
ultMultiplier: 3.0
ultCost: 30
ultCooldown: 10                  // 10초 쿨타임 (추후 수치 확정 시 재조정)
ultMessages: ['강한 내려치기!', '묵직한 일격이 내리꽂힌다!']
```

**초(招) 단계:**

#### 1초 — 강한 내려치기 (심득 80 발견 / 150 해금 / 포인트 3)

절초 해금. 내력이 충분하고 쿨타임이 돌아왔을 때 자동 발동.
아직 초기 단계의 절초라 배율은 3.0, 쿨타임 10초.

```
발견 조건: 삼재검법 심득 80 이상
해금 조건: 삼재검법 심득 150 이상
포인트: 3
```

### 7.3 삼재심법 (심법)

**기본 정보:**
```

faction: 'neutral'
artType: 'simbeop'
role: 'simbeop'

// normalMultiplierCap: 없음 (심법은 초식 미사용)

// ultMultiplier/ultCost/ultCooldown: 없음 (심법은 절초 미사용)
```

**기본 효과 (growth):**
```
growth: {
  baseQiPerSec: 1.0              // 심법 기운 생산 +1/초 시작
  maxQiPerSec: 3.0               // 심득 성장으로 최대 +3/초
  // qiGrowthRate 생략 → BALANCE_PARAMS.QI_GROWTH_RATE (0.075) 사용

  // 전투 수련 (2초 해금 후 활성화)
  // baseCombatQiRatio 생략 → BALANCE_PARAMS.COMBAT_QI_BASE (0.05)
  // combatQiGrowthRate 생략 → BALANCE_PARAMS.COMBAT_QI_GROWTH_RATE (0.006)
  // maxCombatQiRatio 생략 → BALANCE_PARAMS.COMBAT_QI_CAP (0.25)
}
```

**기운 생산 성장 구체 공식 (심법 기여분):**
```
심법qiPerSec = growth.baseQiPerSec + (growth.qiGrowthRate ?? QI_GROWTH_RATE) × sqrt(삼재심법 심득)
             = 1.0 + 0.075 × sqrt(삼재심법 심득), 캡 growth.maxQiPerSec (3.0)
```
- 심득 100: 1.75/초
- 심득 400: 2.5/초
- 심득 ~711: 3.0/초 캡 도달

**총 기운 생산 (calcQiPerSec):**
```
calcQiPerSec = BASE_QI_PER_SEC + 심법qiPerSec + 해금된 MasteryDef의 bonusQiPerSec 합산
```
- 시스템 기본: `BASE_QI_PER_SEC` (1/초)
- 심법 캡 도달 시: 1 + 3.0 = 4.0/초
- 삼재 조화(+2/초) 포함 시: 1 + 3.0 + 2.0 = 6.0/초 (최대)

### 8.4 simulateTick 변경 핵심

```typescript
// 전투 경과 시간 추적
currentBattleDuration += dt

// 내력 회복
stamina = min(stamina + effectiveRegen × dt, maxStamina)

// 절초 쿨타임 감소
if (ultCooldownTimer > 0) ultCooldownTimer -= dt

// 공격 타이밍
if (주공 있고 && stamina >= 주공.ultCost && ultCooldownTimer <= 0 && ultUnlocked) {
  // 절초 발동
  let ultMult = 주공.ultMultiplier
  // 해금된 MasteryDef 중 ultChange가 있으면 심 보정 적용
  const ultChange = getActiveUltChange(state)  // 해금된 mastery에서 ultChange를 찾음
  if (ultChange?.simBonusW) {
    ultMult += ultChange.simBonusW × sim / (sim + ultChange.simBonusH)
  }
  let damage = ATK × ultMult
  stamina -= 주공.ultCost
  ultCooldownTimer = 주공.ultCooldown
} else {
  // 가중치 랜덤 초식
  const art = 선택된무공
  const growthRate = art.growth.normalGrowthRate ?? NORMAL_GROWTH_RATE
  const capIncrease = 해금된 MasteryDef의 normalMultiplierCapIncrease 합산
  const effectiveCap = art.normalMultiplierCap + capIncrease
  const normalMult = min(
    art.growth.baseNormalMultiplier + growthRate × sqrt(해당무공심득),
    effectiveCap
  )
  damage = ATK × normalMult
  // 무공 없으면 ATK × BARE_HAND_MULTIPLIER
}

// 치명타 판정
if (Math.random() < calcCritRate(state)) {
  damage *= calcCritDmg(sim) / 100
}

// 피격 판정
if (Math.random() < calcDodge(state)) {
  // 회피
} else {
  const dmgReduction = calcDmgReduction(state)
  hp -= enemyAtk × (1 - dmgReduction / 100)
  // dmgReduction이 음수면 피해 증가 (의도된 디자인)
}
```

### 8.5 전투 종료 시
```typescript
if (battleMode가 none으로 변경될 때) {
  stamina = 0
  ultCooldownTimer = 0
  currentBattleDuration = 0
}
```

### 8.6 삼재심법 4초 — 처치 시 기운 획득
```typescript
// 적 처치 시
if (killBonusEnabled) {  // 해금된 MasteryDef에 killBonusEnabled가 있는지
  const combatQiRate = calcQiPerSec(state) × calcCombatQiRatio(state)
  const bonusQi = combatQiRate × currentBattleDuration × KILL_BONUS_RATIO
  qi += bonusQi
}
// currentBattleDuration은 다음 적 스폰 시 0으로 리셋
```

`calcQiPerSec`과 `calcCombatQiRatio`가 각각 해금된 MasteryDef의 bonusQiPerSec, bonusCombatQiRatio를 합산하므로, 삼재 조화(+2/초, +10%)는 데이터에서 자동 반영된다.
